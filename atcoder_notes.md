# C++ & AtCoder 学習ノート

## 1. 文字列検索: find
文字列 $t$ の中から部分文字列 $s$ を探す関数。

- **使い方:** `t.find(s)`
- **戻り値:**
    - 見つかった場合：その開始位置のインデックス（$0, 1, 2, \dots$）
    - 見つからない場合：[npos](#2-不一致のシグナル-npos) を返す

### 実装例：隣り合う判定
```cpp
string t = "ACEGBDFAC";
string s; cin >> s;

// 見つかればYes、nposならNoを出力
cout << (t.find(s) != string::npos ? "Yes" : "No") << endl;
```

---

## 2.末尾から検索する：rfind
文字列の中から、指定した文字や文字列が「最後に現れる位置」を検索する機能。

- 使い方: `s.rfind(target)`
- 数学的イメージ: 文字列をインデックスの有限集合 $I$ としたとき、条件を満たすインデックス $i \in I$ の最大値を返す写像。

### 実装例：特定の文字が最後に現れる位置を求める
```cpp
string s = "abracadabra";
// 'a' を末尾から検索（最後に見つかるのはインデックス 10）
auto pos = s.rfind('a');

if (pos != string::npos) {
    // 見つかった場合、そのインデックスを表示
    cout << pos << endl; 
} else {
    // 見つからなければ [npos](#npos) と比較して判定
    cout << -1 << endl;
}
```
---

## 3. 不一致のシグナル: npos
`find` メソッドなどが「何も見つからなかった」時に返す特別な定数。

- **定義:** `static const size_t npos = -1;`
- **正体:** `size_t` 型（符号なし整数）の最大値。
- **注意点:** `int` 型の変数で戻り値を受け取ると、正しく比較できない場合がある。
- **参照:** [find関数の戻り値](#1-文字列検索-find)として利用される。

## 4. 特定の条件の要素を消す: erase-remove
`std::vector` などから、特定の条件に合う要素を「効率的に」削除する定番の手法。

- **使い方:** `v.erase(remove(v.begin(), v.end(), value), v.end());`
- **数学的イメージ:** 1. `remove`: 消したい要素を末尾に追いやり、生きてる要素だけの「新しい境界線」のイテレータを返す（写像の像を左に寄せる操作）。
  2. `erase`: 境界線から末尾までの「ゴミ」を物理的に削除する。

### 実装例：値が 3 の要素をすべて消す
```cpp
vector<int> v = {1, 3, 2, 3, 4};
// 3を末尾に寄せて、ゴミの開始地点から末尾まで削除
v.erase(remove(v.begin(), v.end(), 3), v.end());
// 結果: {1, 2, 4}
```

## 要素の削除と末尾への追加: erase / push_back
std::vector などのシーケンスコンテナにおいて、指定した位置の要素を物理的に削除する機能（erase）と、コンテナの末尾に新しい要素を追加する機能（push_back）。これらを組み合わせることで、配列全体をシフトするような操作が可能になる。

- 使い方: a.erase(a.begin()); / a.push_back(0);
- 数学的イメージ: 有限数列 $A = (a_0, a_1, \dots, a_{n-1})$ に対し、特定の元を取り除いて添字を詰める操作（erase）と、末尾に新たな元 $x$ を結合して $A' = (A, x)$ を構築する操作（push_back）。

### 実装例：先頭の要素を消し、末尾に0を追加する
```cpp
vector<int> a = {10, 20, 30, 40};

// 先頭の要素(10)を削除。残りの要素は自動的に前へ詰められる
a.erase(a.begin()); 

// 末尾に新しい値(0)を追加
a.push_back(0); 

// 結果: {20, 30, 40, 0}
```

## 5. キーと値の対応付け: std::map による頻度集計
連想配列（map）を用いて、特定の要素が何回現れたかを記録・管理する手法。

- 概要: 文字や文字列などの「キー」に対して、出現回数という「値」を紐付けて保持し、動的に更新する。
- 数学的イメージ: 写像 $f: A \to \mathbb{Z}$ において、各要素 $x \in A$ に対応する値 $f(x)$ を逐次加算し、集合 $A$ 上の頻度分布を構築する操作。

### 実装例：文字列に含まれる各文字の頻度を数え上げる
```cpp
string s;
cin >> s;
map<char, int> mp;

// 文字を走査し、対応する整数の値をインクリメント
for (auto &nx : s) {
    mp[nx]++;
}
```

## 6. 順列の全列挙: next_permutation
与えられた要素の並び（順列）を辞書順で「次のパターン」に並べ替える関数。

- 概要: 現在の範囲を辞書順で一つ後ろの順列に変換する。全パターンを網羅するには、事前に範囲を [sort](#sort) して最小の順列にしておく必要がある。
- 数学的イメージ: 有限集合 $S$ の置換 $n!$ 通りを、辞書式順序という全順序に基づいて全走査する操作。

### 実装例：異なる順列をすべて生成し、その総数を求める
```cpp
string S = "aba";
sort(S.begin(), S.end()); // 辞書順最小から開始するためのソート
set<string> s;

do {
    s.insert(S); // 現在の順列を集合に記録
} while (next_permutation(S.begin(), S.end()));

// 結果: s.size() は重複を除いた順列の総数
```

## 7. 最低1回の実行を保証する: do-while文
条件判定をブロックの「末尾」で行うことで、中身の処理を必ず1回は実行させる制御構文。

- 使い方: do { /* 処理 */ } while (条件式);
- 数学的イメージ: 状態 $s$ に対する写像 $f$ の適用を先行させ、遷移後の状態 $f(s)$ が継続集合 $A$ に属するかを事後判定する反復。

### 実装例：[next_permutation](#5) との典型的な併用
```cpp
string s = "abc";
do {
    // 最初の状態（"abc"）を処理した後に、次の順列へ遷移させる
    cout << s << endl;
} while (next_permutation(s.begin(), s.end()));
// 判定が後ろにあるため、最初の並びも漏らさず処理できる
```

## 8. データのペア管理と比較: std::pair
2つの関連する値を一つの「組」としてまとめ、それらを一括で管理・比較する手法。

- 概要: `pair<T1, T2>` を用いて日付や座標などの関連データを一つに束ね、`vector` などのコンテナに格納して走査や検索を行う。値へのアクセスは p.first, p.second
- 数学的イメージ: 直積集合 $M \times D$ の要素 $(m, d)$ を定義し、入力された要素が特定のラベル集合 $G \subset M \times D$ に属するか（$(m, d) \in G$）を判定する操作。

### 実装例：特定のペアが含まれているか検索する
```cpp
int m, d;
cin >> m >> d;

// pair の vector を定義し、あらかじめ対象の集合を用意する
vector<pair<int, int>> gosekkulist = {
    {1, 7}, {3, 3}, {5, 5}, {7, 7}, {9, 9}
};

bool isgosekku = false;
for (const auto &date : gosekkulist) {
    // make_pair や {m, d} で一時的なペアを作り、== で直接比較可能
    if (make_pair(m, d) == date) isgosekku = true;
}

// 判定結果の出力
cout << (isgosekku ? "Yes" : "No") << endl;
```

### 実装例：値と元のインデックスを紐付けて最大値の場所を特定する
```cpp
vector<int> a = {30, 10, 50, 20};
vector<pair<int, int>> b(a.size());

// 第一成分に値、第二成分に元のインデックス（1-based）を持つ元を生成
for (int i = 0; i < a.size(); i++) {
    b[i] = {a[i], i + 1};
}

// 辞書式順序に従いソート（第一成分で比較し、同値なら第二成分で比較）
sort(b.begin(), b.end()); // 関連: [要素の並び替え: sort](#sort)

// 最大値（末尾の元）の第二成分（元のインデックス）を取得
int max_index = b.back().second;
// 結果: 3 (値50のインデックス)
```

## 9. 1-indexed と配列の余白
競技プログラミングなどで、問題文の「1番目」という指定をそのまま配列の添字として扱うために、サイズを少し大きく確保する手法。

- 使い方: `int a[N + 1][M + 1];` （$N \times M$ の行列を扱う場合）
- 数学的イメージ: 集合 $S$ の添字集合 $I = \{0, 1, \dots, n\}$ において、部分集合 $\{1, \dots, n\}$ のみを実運用に割り当てる写像。

### 実装例：2行2列の入力を1ベースで受け取る
```cpp
int r, c;
// 2x2なら a[3][3] で足りるが、余裕を持って a[4][4] とすることもある
int a[4][4]; 

for(int i = 1; i <= 2; i++) {
    for(int j = 1; j <= 2; j++) {
        // 0番目を無視して 1番目, 2番目に直接格納
        cin >> a[i][j];
    }
}
// 入力が (1, 2) なら a[1][2] をそのまま参照できる
cout << a[r][c] << endl;
```

## 10. 文字を小文字に変換する: tolower
英大文字を対応する小文字に変換する機能。アルファベット以外（数字や記号）や既に小文字である場合は、元の値をそのまま返す。

- 数学的イメージ: 写像 $f: C \to C$。大文字の集合を $U \subset C$、小文字の集合を $L \subset C$ とすると、$x \in U$ のとき $f(x) \in L$、それ以外は $f(x) = x$（恒等写像）。

### 実装例：先頭文字を小文字にして接頭辞を加える
```cpp
string s = "Apple";
// 先頭文字 'A' を小文字 'a' に変換
s[0] = tolower(s[0]);
// 文字列を結合
string ans = "an " + s;
// 結果: "an apple"
```

## 11. 特定の要素を別の値に置き換える: replace
std::string や std::vector などのコンテナ内で、特定の値に一致する要素をすべて新しい値に置換する手法。

- 使い方: replace(v.begin(), v.end(), old_value, new_value);
- 数学的イメージ: 特定の元 $a$ を $b$ に移し、それ以外を固定する写像。特定の2元の互換 $(a\ b)$ を実現する場合は、集合外の元 $x$ を経由した合成写像として適用する。

### 実装例：文字列内の '0' と '1' を反転（スワップ）する
```cpp
string s = "1001";
// 一時的な値 'x' を経由した合成写像により、'0' と '1' の互換を実現
replace(s.begin(), s.end(), '0', 'x');
replace(s.begin(), s.end(), '1', '0');
replace(s.begin(), s.end(), 'x', '1');
// 結果: "0110"
```

## 12. 範囲内の最大値の場所を特定する: max_element
指定された範囲の要素を調べ、最大値を持つ「最初の要素へのイテレータ（位置情報）」を取得する手法。

- 使い方: max_element(v.begin(), v.end());
- 数学的イメージ: 全順序集合 $(S, \le)$ において、任意の元 $x$ に対して $x \le m$ を満たす最大の元 $m$ を特定し、元の配列空間におけるその「座標（位置）」を返す写像。

### 実装例：最大値のインデックス（位置）を直接計算する
```cpp
vector<int> a = {10, 50, 20, 50, 30};

// 先頭イテレータとの距離（差分）を計算し、0-basedインデックスを取得
int index_0 = max_element(a.begin(), a.end()) - a.begin();
// 結果: 1 （同値の最大値が複数ある場合、最も先頭に近いものが選ばれる）

// 競技プログラミングで頻出の 1-based インデックスの取得
int index_1 = max_element(a.begin(), a.end()) - a.begin() + 1;
// 結果: 2

// 関連: 実体（値そのもの）が欲しい場合は * 演算子でデリファレンスする
// int max_val = *max_element(a.begin(), a.end());
```

---
### 💡 注釈: なぜ a.begin() を引く必要があるのか？
`max_element` などの標準アルゴリズムは、配列のインデックスではなく**「イテレータ（メモリ上の位置情報）」**を返す仕様になっているため。

- **仕組みのイメージ**: 
  イテレータは「メモリの住所」のようなもの。「最大値の住所」から「先頭の住所（`a.begin()`）」を引き算して、初めて先頭からの「距離」が求まる。
- **インデックスの取得**:
  1. `イテレータ - a.begin()` の計算結果が、そのまま **0-basedのインデックス**（0始まり）になる。
  2. 競技プログラミング等で **1-basedのインデックス**（1番目、2番目…）にする場合は、最後に `+ 1` をする。
- **注意点**:
  イテレータのままでは、数値として `cout` で出力することはできない（コンパイルエラーになる）。

## 13. 最も近い整数に丸める: round
浮動小数点数を最も近い整数に丸める（四捨五入する）手法。競技プログラミングでは、計算誤差を吸収するために微小な値（$\epsilon$）を加えてから処理するテクニックがよく使われる。

- 使い方: round(value); （戻り値は浮動小数点数型であるため、通常は int 等にキャストして用いる）
- 数学的イメージ: 実数 $x$ を最も近い整数に写す写像。半整数（小数部分がちょうど $0.5$）の場合は絶対値が大きくなる方向へ丸められる（$x \ge 0$ においては $\lfloor x + 0.5 \rfloor$ と同値）。

### 実装例：浮動小数点数の誤差を考慮した丸め処理
```cpp
double a = 2.4999; // 本来は 2.5 になるべき計算結果（誤差を含むと仮定）

// 通常の丸め（このままだと2.4999は2に丸められてしまう）
int normal_ans = (int)round(a); // 結果: 2

// 微小な値（0.0005など）を足すことで、誤差による意図しない切り捨てを防ぐ
int safe_ans = (int)round(a + 0.0005);
// 結果: 3
```

## 14. 特定の形式に従って入力を抽出する: scanf
指定したフォーマット（形式）文字列に従って標準入力からデータを読み取り、変数に直接代入するC言語由来の手法。競技プログラミングにおいて、特定の文字（`.` や `/` など）を区切りとして読み飛ばしながら数値を抽出する際に極めて有効。

- 使い方: scanf("フォーマット文字列", &変数1, &変数2, ...);
- 数学的イメージ: 入力された文字列の集合 $\Sigma^*$ に対し、フォーマット指定子が定義する構造とのパターンマッチングを行い、抽出した部分文字列を目的のデータ型の空間（例: $\mathbb{Z} \times \mathbb{Z}$）へ直接写す射影。

### 実装例：小数点を区切り文字として整数部と小数部を分割する
```cpp
int a, b;
// 入力例 "3.14" において、'.' を単なる区切り（無視する文字）として扱い、
// 3 を a に、14 を b に整数として抽出・代入する。
// ※代入先の変数はメモリアドレスを渡す必要があるため '&' が必須
scanf("%d.%d", &a, &b);

// 小数第一位の値を使って四捨五入を判定するロジックの例
// 浮動小数点数（double等）の丸め誤差を完全に回避できる利点がある
if (to_string(b)[0] >= '5') {
    ++a;
}
```
---
### 💡 注釈: これは scanf (C言語スタイル) 特有の手間ですか？
はい、その通りです。`char S[3];` のように手動で「文字数＋1」の余裕を計算・確保するのは、**C言語由来の関数（`scanf` や `printf` など）で文字配列を扱う際などに特有の書き方**です。

- **C言語スタイル (`char` 配列 + `scanf`)**:
  - コンピュータは「用意された箱（メモリ）」に直接データを流し込むだけなので、人間が事前に十分なサイズの箱を用意し、終端（ヌル文字 `\0`）のスペースまで管理する責任を持ちます。
- **C++スタイル (`std::string` + `cin`)**:
  - `string` という便利なクラスがメモリ管理を全自動で行ってくれます。
  - `string S; cin >> S;` と書くだけで、入力された長さに合わせて裏側で勝手に箱を広げ、ヌル文字の処理も隠蔽してくれるため、人間がサイズを気にする必要はありません。

競技プログラミングでは、極限の実行速度を求めてあえてC言語の `scanf` と `char` 配列を使う層もいますが、バグを減らしたい場合は `std::string` と `cin` を使うのが現代C++の基本スタイルです。

## 15. ビット演算で大文字・小文字を反転する: ^= 32
ASCIIコードの規則性を利用し、排他的論理和（XOR）を用いてアルファベットの大文字と小文字を高速に相互変換（反転）する手法。

- 使い方: c ^= 32;
- 数学的イメージ: アルファベットの文字集合に対する対合（インボリューション） $f(x) = x \oplus 32$ を適用し、大文字の集合と小文字の集合との間で双方向の全単射を構成する。

### 実装例：条件を満たしたときに先頭文字の大小を反転させる
```cpp
char S[3] = "Y";
char T[3] = "abc";

// Sの先頭が 'Y' であるか判定
if (S[0] == 'Y') {
    // ASCIIコードにおいて大文字と小文字はちょうど 32 (2^5) 離れている。
    // 32とのXOR演算により特定ビットのみを反転させ、'a' を 'A' に変換する
    T[0] ^= 32;
}
// 結果: T は "Abc" になる
// 関連: 入力の抽出 [scanf](#14)
```
---
### 💡 注釈: なぜ `^= 32` で大文字・小文字が反転するのか？
アルファベットの「ASCIIコード（コンピュータ内部での文字の背番号）」の2進数表現における美しい規則性を利用しているためです。

- 'A' の番号は 65 （2進数で `01000001`）
- 'a' の番号は 97 （2進数で `01100001`）

大文字と小文字の番号は**常に 32 離れています**。
これを2進数で見ると、右から6番目のビット（$2^5 = 32$ の位）が `0` なら大文字、`1` なら小文字という規則になっています。
XOR（排他的論理和 `^`）は「指定した箇所のビットだけを反転させる」という性質を持つため、`32`（2進数で `00100000`）とXOR演算をすることで、**文字の種類（1〜5番目のビット）は維持したまま、大文字・小文字のフラグだけを綺麗に切り替える**ことができるのです。

## 16. 特定の形式で値を出力する: printf
指定したフォーマット（形式）文字列の「穴（指定子）」に変数を埋め込み、標準出力へ書き出すC言語由来の手法。`cout` と比較して実行速度が速い場合が多く、競技プログラミングでも好まれる。

- 使い方: printf("フォーマット文字列", 変数1, 変数2, ...);
- 数学的イメージ: 文字列のテンプレート（型）に対して、変数の直積空間から値を代入し、具体的な出力文字列へと写す射影。

### 実装例：文字列を埋め込んで改行する
```cpp
char T[] = "Abc";

// %s は「文字列（char配列など）」を埋め込むためのフォーマット指定子
// \n は「改行」を表す特殊文字
printf("%s\n", T);
// 結果: Abc と出力された後、改行される

// 関連: フォーマットに従った入力 [scanf](#14)

```

## 17. 重複の排除と種類数の判定: std::set
複数の値から重複を自動的に排除し、異なる要素が「何種類あるか」を簡潔に判定する手法。

- 使い方: set<int> S({A, B, C});
- 数学的イメージ: 与えられた多重集合から重複元を縮約して純粋な集合 $S$ を構築し、その濃度（要素数） $|S|$ を評価する。

### 実装例：3つの変数が「ちょうど2種類」であるかを判定
```cpp
int A = 5, B = 7, C = 5;
```

## 18. 集合への要素の追加: insert
std::set などの連想コンテナに新しい要素を追加する手法。すでに同じ値が存在する場合は自動的に無視される。

- 使い方: s.insert(value);
- 数学的イメージ: 既存の集合 $S$ に新たな元 $x$ を追加し、和集合 $S \cup \{x\}$ を構築する。すでに $x \in S$ である場合、集合の構成と濃度は変化しない。

std::vector などのシーケンスコンテナにおいて、イテレータを用いて指定した特定の位置（インデックス）に新しい要素を割り込ませる手法。

- 使い方: v.insert(v.begin() + K, X);
- 数学的イメージ: 有限数列 $A = (a_0, a_1, \dots, a_{n-1})$ の第 $k$ 添字位置に元 $x$ を挿入し、要素が右へシフトした新たな数列 $A' = (a_0, \dots, a_{k-1}, x, a_k, \dots, a_{n-1})$ を構築する操作。

### 実装例：値を1つずつ追加して種類数を数える
```cpp
set<int> s;

s.insert(5);
s.insert(7);
s.insert(5); // 既に 5 が存在するため追加されない

// 結果: s の要素は {5, 7} のみとなる
if (s.size() == 2) {
    // 追加された一意な値がちょうど2種類だった場合の処理
}
```

### 実装例：配列のインデックス K の位置に値 X を挿入する
```cpp
vector<int> A = {10, 20, 30};
int K = 1;  // 挿入したいインデックス
int X = 15; // 挿入する値

// 先頭イテレータに K を足して場所を指定する
A.insert(A.begin() + K, X);
// 結果: {10, 15, 20, 30}
```

## 20. 符号付き整数型とマクロ置換の回避: signed
単独で記述された場合は通常の `int`（符号付き整数）と完全に同義となる型指定子。競技プログラミングにおいて `#define int long long` というオーバーフロー対策のマクロを組む際、標準規格で戻り値が `int` でなければならない `main` 関数のコンパイルエラーを回避する目的で、置換対象にならない `signed main()` として使われる（提示コードではそのテンプレートの名残として書かれている）。

```cpp
}
```

## 21. 文字列の長さの取得: length / size
std::string 型の変数が保持する文字列の長さ（文字数）を取得するメソッド。結論から言うと、`length()` と `size()` の機能は「全く同じ」である（前者は文字列としての直感的な表現、後者は他のSTLコンテナとの統一性を保つために両方用意されている）。

```cpp
string S = "bus";

// 長さを利用して末尾の文字（インデックスは len - 1）にアクセスする
if (S[len - 1] == 's') {
    S += "es";
} else {
    S += "s";
}
// 結果: "buses"

```

## 23. 要素の巡回シフト: rotate
要素の並び順を保ったまま、指定した位置の要素が新たな「先頭」となるように、配列全体をローテーション（回転）させるアルゴリズム。

- 使い方: rotate(a.begin(), a.begin() + m, a.end());
- 数学的イメージ: 有限数列 $A = (a_0, \dots, a_{n-1})$ の第 $m$ 要素を新たな先頭とし、各要素の添字に対して巡回群の元を作用させる合同式 $i \mapsto (i - m \bmod n)$ によって列を再構成する。

### 実装例：末尾のK個を先頭に移動させる（右シフト）
```cpp
vector<int> a = {10, 20, 30, 40, 50};
int n = 5;
int k = 2; // 後ろから2つの要素を前に持ってくる

// a.begin() + 3 （値が 40 の位置）が新たな先頭になるように回転させる
rotate(a.begin(), a.begin() + n - k, a.end());

// 結果: {40, 50, 10, 20, 30}

```

## 24. 64ビット整数型の定数指定: 1LL
プログラム中の数値「1」を、標準の32ビット整数（int）ではなく64ビット整数（long long）として明示的に扱うための接尾辞。主にビットシフト演算などで巨大な数を扱う際の桁あふれ（オーバーフロー）を防ぐ役割を持つ。

```cpp
long long height = 1LL << day;
```

- 数学的イメージ: 文字集合 $\Sigma$ において、文字 $c$ が特定の部分集合（大文字集合または小文字集合）に属するかを真理値に写像する指示関数 $\chi: \Sigma \to \{0, 1\}$。

```cpp
for (int i = 1; i < S.size(); ++i) {
```

## 26. 降順（大きい順）ソートの指定: greater

## 27. 連続区間のXOR和と欠落要素の探索: 排他的論理和の性質
連続する整数の配列から「欠けている1つの数字」を高速かつ省メモリで特定するアルゴリズム。同じ数を2回XORすると0になる性質と、「4の倍数から始まる連続する4つの整数のXOR和は常に0になる（例: 4^5^6^7 = 0）」という周期性を利用している。

- 概要: 入力された数値群のXOR和と、あるべき完全な区間 $[min, max]$ のXOR和をぶつけることで、ペアにならない「抜けている数字」だけをあぶり出す。
- 数学的イメージ: 連続区間の完全な集合 $S = [m, M]$ と、1要素が欠落した集合 $A = S \setminus \{x\}$ において、$\left(\bigoplus_{s \in S} s\right) \oplus \left(\bigoplus_{a \in A} a\right)$ を評価し、群の対合性 $y \oplus y = 0$ により欠落元 $x$ を同定する。

### 実装例：連続する数値から抜けている1つを見つける
```cpp
int min_A = 1000, max_A = 0, xor_A = 0;

// 1. 入力値の最小値、最大値、および入力全体のXOR和を求める
for (int A : input_array) {
    min_A = min(A, min_A);
    max_A = max(A, max_A);
    xor_A ^= A;
}

// 2. [0, max_A] のXOR和を xor_A に適用する
// （4の倍数未満のXOR和は0になるため、そこまで逆算すればよい）
++max_A; 
while (max_A % 4 != 0) xor_A ^= --max_A;

// 3. [0, min_A - 1] のXOR和を xor_A に適用して差し引く
// （これで完全な区間 [min_A, max_A] のXOR和をぶつけたことになる）
while (min_A % 4 != 0) xor_A ^= --min_A;

// 結果: xor_A には欠落していた数値が残る
```

## 29. 先入れ先出しのデータ構造: queue
データを追加した順に取り出す「待ち行列（FIFO）」を管理するコンテナ。`std::vector`でも似た処理は可能だが、`vector`の先頭要素の削除は後ろの要素をすべて前に詰めるため計算量が O(N) かかり致命的に遅くなる。一方、`queue`の先頭削除は O(1) で完了するため、順番待ちの処理において極めて効率的である。

- 使い方: `q.push(value)`で末尾に追加し、`q.front()`で先頭を取得、`q.pop()`で先頭を削除する。
- 数学的イメージ: 時刻によって順序付けられた要素の集合 $S$ に対し、追加操作は常に $\max(S)$ の直後への配置として働き、削除操作は常に $\min(S)$ の要素を取り除く動的な全順序集合。
- 関連: 両端に対する高速な追加・削除が必要な場合は [deque](#deque) を、後入れ先出しが必要な場合は [stack](#stack) を参照。

### 実装例：順番待ちのシミュレーション
```cpp
queue<int> que;

// データの追加（末尾に並ぶ）
que.push(10);
que.push(20);

// 先頭要素の確認と削除（vectorと違い、O(1)で高速に処理）
int first = que.front(); // 10
que.pop();

// 現在の先頭要素を確認
int next = que.front(); // 20
```

## 30. 任意の要素への直接アクセス: ランダムアクセス
`queue` は先頭と末尾以外のデータに一切触れられないという厳しい制約があるため、途中のデータを確認・変更したい場合には使えない。用途に応じて、インデックスを指定してどこへでも O(1) で即座にアクセスできる `std::vector` などの「ランダムアクセス」可能なコンテナと使い分ける必要がある。

- 使い方: `v[index]`
- 数学的イメージ: 有限全順序集合 $S$ に対して、添字集合 $I = \{0, 1, \dots, n-1\}$ からの全単射 $f: I \to S$ が定義され、任意の $i \in I$ について $f(i)$ を定数時間で評価する写像。
- 関連: 先頭要素への順次アクセスに特化したコンテナについては [queue](#queue) を参照。

### 実装例：インデックスを指定した要素の取得と変更
```cpp
vector<int> v = {10, 20, 30, 40};

// queueでは中身の検索やソートができないが、vectorなら任意の場所に直接アクセス可能
int target = v[2]; // O(1)で 30 を取得

// 途中の要素の書き換えも自由自在
v[2] = 100;
// 結果: {10, 20, 100, 40}
```

## 31. ビットごとの論理積（AND）: &
2つの整数を2進数のビット列とみなし、各桁ごとに論理積を計算する演算子。主に、特定の桁のビットが `1` であるかを確認する判定処理（マスク処理）や、特定のビットを `0` にクリアする目的で使用される。

- 使い方: `a & b`
- 数学的イメージ: 整数を底2の展開係数の数列（ビット列）とみなし、各桁ごとに二元体 $\mathbb{F}_2$ 上の乗法を適用する写像。

### 実装例：特定のビットが立っている（1である）かの判定
```cpp
int n = 10; // 2進数で 1010 (8 + 2)
int i = 1;  // 下から1番目（0-indexed、つまり「2」の位）を調べたい

// 1 << i は 1 を左に i 回シフト（2進数で 0010 となる）
// 1010 & 0010 を計算すると 0010 となり、非ゼロなので条件式は true となる
if (n & (1 << i)) {
    // nの i 番目のビットは 1 である
}
```

## 32. ハッシュ探索と右端のゼロカウント: unordered / countr_zero
`unordered`（`unordered_set`や`unordered_map`など）はハッシュテーブルを用いて順序を保持せずに要素を O(1) で高速に管理するコンテナ群。`countr_zero` は整数の2進数表現において、一番右（下位）から連続する `0` のビット数を数える関数である。

- 使い方: `unordered_set<型> 変数` / `countr_zero(値)`
- 数学的イメージ: `unordered`はハッシュ関数を用いた定数時間の元判定・対応付けであり、`countr_zero`は整数 $N$ を $2^k \cdot m$ ($m$は奇数) と素因数分解した際の指数 $k$ を抽出する写像。
- 関連: 要素への定数時間アクセスを持たない順序コンテナについては [set/map](#set_map) を、ビット判定の基礎は [ビットごとの論理積（AND）](#31) を参照。

### 実装例：ハッシュを用いた管理と末尾ゼロの取得
```cpp
// --- countr_zero の利用 ---
unsigned n = 24; // 2進数で 11000
// 右から続く0は3つなので、3 を返す
int zero_bits = countr_zero(n);

// --- unordered_set の利用 ---
unordered_set<int> us = {10, 20, 30};
us.insert(40);

// vectorでの検索は O(N) かかるが、unordered_set なら O(1) で即座に判定可能
if (us.count(20)) {
    // 20は存在する
}

```

## 33. 符号なし整数型: unsigned

```cpp
unsigned int underflow = -1;

// for (unsigned i = 5; i >= 0; --i) { ... }

```

## 34. 条件を満たす境界の高速探索: 二分探索（Binary Search）
単調に変化する事象に対して、1から順番に調べるのではなく、常に探索範囲の中央を評価して範囲を半分に絞り込んでいくことで、条件を満たす値を $O(\log N)$ で極めて高速に見つける手法。`1e9`（$10^9$）のような巨大な範囲が対象でも、約30回のループで特定できる。

- 使い方: 左端 `lef` と右端 `rig` を定義し、中央値 `mid` での判定結果に応じて `lef` または `rig` を更新する。
- 数学的イメージ: 狭義単調増加関数 $f(x)$ と定数 $k$ が与えられたとき、$f(x) \ge k$ を満たす最小の $x$ を、定義域の区間縮小法により特定するアルゴリズム。
- 関連: 巨大な数値の簡略表記には `1e9`（$10^9$）などの浮動小数点数リテラルが頻繁に用いられる。

### 実装例：合計が k 以上になる最小の回数を探す
```cpp
ll n = 5, k = 50;
ll lef = 0, rig = 1e9; // 1e9 は 1000000000 と同義

// 探索範囲が逆転するまで繰り返す
while (lef <= rig) {
    ll mid = (lef + rig) / 2;
    // 等差数列の和：(項数 * (初項 + 末項)) / 2
    ll cur = ((mid + 1) * (n + (n + mid))) / 2;
    
    if (cur < k) {
        // 合計が k 未満なら、もっと大きな mid が必要
        lef = mid + 1;
    } else {
        // 合計が k 以上なら、さらに小さな mid でも条件を満たすか試す
        rig = mid - 1;
    }
}
// 最終的に lef が「条件を満たす最小の mid」となる
```

## 36. ビットごとの排他的論理和（XOR）: ^
2つの整数を2進数のビット列とみなし、各桁の値が「異なる場合のみ `1`、同じ場合は `0`」とする演算子。「同じ値を2回XORすると `0` になる（相殺される）」「順序を入れ替えても結果が同じ（交換法則）」という性質を持ち、ペアになっていない単独の要素を抽出したり、状態を反転させたりする用途で非常に強力である。

- 使い方: `a ^ b`
- 数学的イメージ: 整数を二元体 $\mathbb{F}_2$ 上のベクトル空間とみなし、各成分について加法（自己単逆性 $x+x=0$）を適用する写像。
- 関連: 特定のビットが `1` かどうかの判定・抽出については [ビットごとの論理積（AND）](#31) を参照。

### 実装例：ペアになっていない値（長方形の残りの頂点など）を見つける
```cpp
int x1 = 10, x2 = 20, x3 = 10;

// 演算の順序に関係なく、同じ値(10)同士はXORで 0 に相殺される
// 最終的に 0 ^ 20 となり、ペアのない値(20)だけが抽出される
int missing_x = x1 ^ x2 ^ x3;
// 結果: 20
```

## 37. 要素の集約と畳み込み: accumulate
配列などの指定した範囲の要素を、初期値に対して順番に演算（デフォルトは足し算）して1つの値にまとめる関数。第4引数にカスタムの計算ルール（ラムダ式など）を渡すことで、掛け算やXORなど、任意の二項演算で集約（リデュース）を行うことができる。

- 使い方: `accumulate(開始位置, 終了位置, 初期値, [カスタム演算])`
- 数学的イメージ: 集合の要素列 $(a_1, a_2, \dots, a_n)$ と初期値 $v_0$、任意の二項演算 $\circ$ に対して、$v_k = v_{k-1} \circ a_k$ を順次評価して最終的な値 $v_n$ を得る左畳み込み（Left Fold）写像。
- 関連: カスタム演算の定義に使われているXORの性質については [ビットごとの排他的論理和（XOR）](#36) を参照。

### 実装例：合計値の計算と、カスタム演算（XOR和）の適用
```cpp
vector<int> a = {1, 2, 3, 4, 5};

// デフォルト（足し算）: 初期値0に対し、すべての要素を足す
int sum = accumulate(a.begin(), a.end(), 0);
// 結果: 15

// カスタム（XOR）: 初期値0に対し、すべての要素を順にXOR演算でまとめる
int xor_sum = accumulate(a.begin(), a.end(), 0, [](int x, int y) { return x ^ y; });
```

## 38. 任意の条件で並び替える: sortのカスタム比較（ラムダ式）
`std::sort` の第3引数に独自の比較ルールを渡すことで、デフォルトの昇順や辞書順とは異なる並び替えを実現する手法。`[](型 x, 型 y) { 条件式 }` という構文（ラムダ式、名前のない使い捨ての関数）を用いて、「左の要素 `x` が右の要素 `y` より前に並ぶべきなら `true` を返す」というルールをその場で定義している。ご提示のコードでは「文字列が短い順」にソートされる。

- 使い方: `sort(開始, 終了, [](型 x, 型 y) { return 比較条件; });`
- 数学的イメージ: 集合 $S$ における要素間の順序関係（狭義弱順序）を、要素自身の値ではなく、任意の評価関数 $f(x)$ に基づく関係 $f(x) < f(y)$ として動的に再定義する操作。
- 関連: ラムダ式を利用した別の集約処理については [accumulate](#37) を参照。

### 実装例：文字列を「文字数が少ない順」にソートする
```cpp
vector<string> s = {"apple", "kiwi", "banana"};

// デフォルトのアルファベット順ではなく、長さを基準に比較する
sort(s.begin(), s.end(), [](string x, string y) {
    return x.size() < y.size();
});
// 結果: {"kiwi", "apple", "banana"}
```

## 39. 最大・最小値を常に先頭に保つ: priority_queue
データを追加するたびに自動で並び替えを行い、常に最大（または最小）の要素を O(1) で取り出せるコンテナ。要素の追加や削除のたびに `vector` を `sort` すると計算量が $O(N \log N)$ かかるが、`priority_queue` は内部でヒープ（二分木）を維持するため、追加・削除を $O(\log N)$ で完了でき圧倒的に高速である。

- 使い方: `pq.push(value)` で追加、`pq.top()` で先頭を取得、`pq.pop()` で先頭を削除。
- 数学的イメージ: 集合の要素が完全二分木のノードへ写像され、任意の親ノードがその子ノードよりも常に大きい（または小さい）という半順序関係（ヒープ条件）を動的に維持する構造。
- 関連: 単純な先入れ先出し（FIFO）のデータ構造については [queue](#29) を参照。

### 実装例：最小値を常に取り出すキュー（Min-Heap）
```cpp
// デフォルトは最大値が先頭だが、greater<> を指定すると最小値が先頭になる
priority_queue<int, vector<int>, greater<int>> pq;

pq.push(30);
pq.push(10);
pq.push(20);

// vectorを毎回sortしなくても、O(1)で最小値にアクセスできる
int smallest = pq.top(); // 10

// O(log N) で10を削除し、内部構造を再構築する（次は20が先頭になる）
pq.pop(); 
```

- 数学的イメージ: 外部からの入力ストリームを特定のパターンで解析し、抽出された値をポインタ空間上の指定された座標（アドレス）が指す記憶領域へ直接写像する操作。

## 41. 基準位置による高速な分割: nth_element
指定した位置の要素だけを「全体をソートした際に来るべき正しい値」にし、その前後に「基準より小さい値の集合」と「大きい値の集合」を分割する関数。全体を並び替える `sort` が O(N log N) の時間を要するのに対し、内部的にクイックソートの分割工程だけを行う「クイックセレクト」というアルゴリズムを用いるため、平均 O(N) で極めて高速に完了する。提示されたコードはこれを利用し、上位と下位の極端な値（それぞれN個）を高速に切り捨てて、残った中間層の平均（トリム平均）を計算している。

- 使い方: `nth_element(開始位置, 基準となる位置, 終了位置)`
- 数学的イメージ: 全順序集合の要素列 $A$ と添字 $k$ に対し、$A[k]$ を完全ソート時と同値に固定し、$\forall i < k, \forall j > k$ について $A[i] \le A[k] \le A[j]$ となるよう部分集合を再配置する写像。
- 関連: 全体の並び替えについては [sortのカスタム比較](#38) を、要素の合計を求める `reduce`（`accumulate`の並列実行を許容する版）については [accumulate](#37) を参照。

### 実装例：上位・下位の順序は無視して、中央値（メジアン）だけを高速に取得する
```cpp
vector<int> v = {9, 2, 7, 3, 1, 8, 4, 6, 5};

// 真ん中の位置（インデックス4）を基準に配列を分割する
nth_element(v.begin(), v.begin() + 4, v.end());

// v[4] には確実に中央値である '5' が入る
// 前半（例: 4, 2, 1, 3）や後半（例: 8, 9, 7, 6）の内部の順序はバラバラのままになる
int median = v[4];
```

## 42. パターンマッチによる柔軟な置換: regex_replace
正規表現（パターン）を用いて、条件に一致する部分文字列をすべて別の文字列に置換する関数。単一の文字（`'a'` を `'o'` など）しか置換できない `<algorithm>` の `std::replace` や、事前に位置（インデックス）と長さを自力で特定する必要がある `std::string::replace` と比べ、圧倒的に柔軟な検索と一括置換が可能である。ただし、正規表現エンジンの起動を伴うため実行速度は比較的遅い。

- 使い方: `regex_replace(対象文字列, regex("パターン"), "置換後の文字列")`
- 数学的イメージ: 文字列（文字の有限列）の集合において、正規言語が受理する部分列を特定し、それらを別の部分列へと一斉に写像する文字列変換操作。
- 関連: 特定の条件を満たす要素を取り除く操作については [erase-remove](#3) を参照。

### 実装例：特定の文字列パターンを一括置換する
```cpp
string s = "banana";

// 比較: std::replace は単一の「文字(char)」しか扱えない
// replace(s.begin(), s.end(), 'a', 'o'); // "bonono" になる

// regex_replace は「文字列のパターン」を検索して一気に置換できる
// "na" という並びをすべて "nya" に置き換える
string result = regex_replace(s, regex("na"), "nya");
// 結果: "banyanya"
```

## 正規表現による置換と非重複マッチ: regex_replace
`std::regex_replace` で "TT" を "PC" に置換する際、"TTT" のような連続部分でエラーや意図しない重複適用は起きないという性質と、競技プログラミングにおける計算量上の注意点。

- 概要: 正規表現は「マッチした文字を消費」しながら左から右へ探索を進めるため、"TTT" は最初の2文字で "PC" を作り、残りの1文字 "T" はスルーされる。これは「操作回数の最大化（左から貪欲に取るのが最適）」という題意と完全に一致する。
- 数学的イメージ: 文字列 $S$ から互いに素（disjoint）な隣接要素の組 $(S_i, S_{i+1}) = (T, T)$ を左から貪欲に抽出して写像する操作であり、インデックスの重なり合いは自動的に排除される。

### 実装例：TTT の置換と、より高速な代替案
```cpp
string s = "TTT";
// エラーにはならず、安全に "PCT" に変換される。題意を満たす正当な操作。
s = regex_replace(s, regex("TT"), "PC");

// 【重要】競技プログラミングにおける注意
// std::regex_replace は内部の定数倍が非常に重く、文字列が長いと TLE（実行時間超過）の危険がある。
// 以下のシンプルなループの方が圧倒的に高速であり、実戦で推奨される。
for (int i = 0; i < (int)s.size() - 1; ++i) {
    if (s[i] == 'T' && s[i+1] == 'T') {
        s[i] = 'P';
        s[i+1] = 'C';
        // s[i+1] が 'C' に書き換わるため、次のループ(i+1)で再評価されても
        // 重複してPCが作られることはなく、"TTT" は正しく "PCT" になる。
    }
}
```

## 43. 巧みな空白と改行の分岐出力: " \n"[条件]
条件分岐（`if`文）をわざわざ書かずに、配列の最後の要素の後には改行を、それ以外の要素の後には空白を出力する、競技プログラミング特有のトリッキーで簡潔な記法。`" \n"` という文字列（文字の配列）に対して、条件式の真偽（`false` なら `0`、`true` なら `1` に変換される）をインデックスとして与えることで、出力する文字を動的に切り替えている。

- 使い方: `" \n"[条件式]`
- 数学的イメージ: 命題 $P$ の真理値 $\{0, 1\}$ を添字として、文字の集合 $\{\text{空白}, \text{改行}\}$ から要素を選択する指示関数（特性関数）の応用。
- 関連: インデックスを指定した値の取得の基礎については [ランダムアクセス](#30) を参照。

### 実装例：末尾のみ改行するループ出力
```cpp
vector<int> ans = {10, 20, 30};
int n = ans.size();

for (int i = 0; i < n; i++) {
    // i + 1 == n が false(0) のときは " \n"[0] すなわち空白(' ')が出力される
    // 最後の要素のときだけ true(1) となり、" \n"[1] すなわち改行('\n')が出力される
    cout << ans[i] << " \n"[i + 1 == n];
}
// 結果: "10 20 30\n"（最後に1回だけ改行される）
```

## 44. 重複要素の高速な排除: unique
配列内の「連続する」重複要素を1つにまとめ、残りの要素を前方に詰める関数。配列全体から重複を消すには事前に `sort` が必要となる。要素の追加時に毎回メモリの動的確保や木の再構築を行う `std::set` に比べ、メモリが連続している `vector` 上で `sort` と `unique` を組み合わせる方が、キャッシュ効率が極めて良く、実際の実行速度（定数倍）が圧倒的に速い。

- 使い方: `v.erase(unique(v.begin(), v.end()), v.end());`
- 数学的イメージ: 全順序集合の有限列において、隣接する同値な要素の連なり（同値類）を単一の代表元へと縮約し、単調増加な部分列を生成する写像。
- 関連: 詰められた後の不要な末尾要素を物理的に削除する仕組み（erase-removeイディオム）については [erase-remove](#3) を参照。

### 実装例：配列からすべての重複要素を削除する
```cpp
vector<int> a = {3, 1, 2, 1, 3};

// uniqueは「連続する」重複しか処理できないため、まずはソートして同じ値を隣接させる
sort(a.begin(), a.end());
// a の状態: {1, 1, 2, 3, 3}

// uniqueで重複を末尾に追いやり、有効な要素の「新しい境界」を取得。その後ろをeraseで削除
a.erase(unique(a.begin(), a.end()), a.end());
// 結果: {1, 2, 3}
```

## 45. 固定長の連続メモリ領域: 生配列（Raw Array）
C言語由来の最も基本的なデータ構造。`std::vector` が必要に応じてサイズを自動拡張（動的確保）し、`size()` などの便利な機能を持つのに対し、生配列は「宣言時に決めたサイズから一切変更できない（固定長）」「便利な関数を持たない」という違いがある。サイズ変更ができない代わりに管理コスト（オーバーヘッド）がなく、ご提示のコードのようにグローバル空間に巨大なサイズ（100000など）を宣言して使い回すことで、実行速度を極限まで高める競技プログラミングのテクニックとしてよく使われる。

- 使い方: `型 変数名[定数サイズ];`
- 数学的イメージ: 添字集合 $I = \{0, 1, \dots, N-1\}$ から値の集合への写像であり、定義域 $I$ の濃度（要素数 $N$）が時間発展において完全に不変である静的な構造。
- 関連: 途中でデータ数が変わる場合や安全なメモリ管理が必要な場合は、動的配列である [ランダムアクセス](#30)（`std::vector`）を参照。

### 実装例：固定長配列の初期化とアクセス
```cpp
// 宣言時に要素数が固定される（後から追加・削除は不可能）
// ※関数の外（グローバル）で宣言すると、自動的にすべて 0 で初期化される
int a[5]; 

// vectorのような push_back() 等の機能はなく、直接インデックスを指定して操作するのみ
a[0] = 10;
a[1] = 20;

// アクセス速度は vector[i] と同様に O(1) で極めて高速
int val = a[0];
```

## 46. 0-indexed変換とグループごとの最大値の抽出
入力された「1始まり（1-based）」の番号を、C++の配列で扱える「0始まり（0-based）」のインデックスに変換するための必須の操作（`a[i]--`）。コード全体としては、同じ番号（グループ）を持つ要素の中から「最大の重み」を持つものを1つだけ残し、それ以外の「捨てられる要素の重みの合計」を（全体の合計 － 最大値の合計）という引き算によって効率的に求めている。

- 使い方: `a[i]--` でインデックスを1減らし、`max(現在の最大値, 新しい値)` で最大値を更新する。
- 数学的イメージ: 要素の添字集合 $I$ を同値関係で類別し、各同値類 $C$ において $\max_{x \in C} w(x)$ を満たす代表元を一つ残して他を補集合として捨てる際の、補集合の重み和の計算。
- 関連: 配列のインデックスが0から始まる仕様については [生配列](#45) や [ランダムアクセス](#30) を参照。

### 実装例：各グループで一番重いものだけを残し、それ以外を捨てるコストの計算
```cpp
// 競技プログラミング等の入力は「1〜N」であることが多いが、C++の配列は「0〜N-1」であるため、
// そのまま a[i] をインデックスとして使うと範囲外アクセス（セグメンテーション違反）になる。
// したがって、使う直前に 1 を引いておく。
a[i]--; 

// グループ a[i] における、これまでの最大重みと今回の重みを比較し、大きい方を保存
max_weight[a[i]] = max(max_weight[a[i]], w[i]);

// 最後に「全体の重みの合計」から「各グループで残した最大重みの合計」を引くことで、
// 捨てた要素の重みの合計がわかる
// sum_w - sum_max
```

## 47. 後入れ先出しのデータ構造: stack
最後に追加したデータを最初に取り出す「スタック（LIFO: Last-In-First-Out）」を管理するコンテナ。机の上に本を積み上げ、上から順番に取っていくような構造を持つ。操作できるのは常に一番上（最新）のデータのみであるため、直近の履歴をたどる処理や、迷路などの「深さ優先探索（DFS）」で頻繁に利用される。

- 使い方: `st.push(value)`で一番上に追加し、`st.top()`で一番上を取得、`st.pop()`で一番上を削除する。
- 数学的イメージ: 時刻によって順序付けられた要素の集合 $S$ に対し、追加・取得・削除のすべての操作が常に $\max(S)$ （最新の要素）に対してのみ作用する動的な全順序集合。
- 関連: 先入れ先出し（FIFO）が必要な順番待ちの処理については [queue](#29) を参照。

### 実装例：直近のデータを取得・削除する
```cpp
stack<int> st;

// データの追加（上に積み上げていく）
st.push(10);
st.push(20);

// 一番上の要素を確認（O(1)で高速に処理）
// 最後に積んだ 20 が得られる
int current_top = st.top(); 

// 一番上の要素を削除（20が消える）
// これにより、次に top() を呼ぶと 10 が得られるようになる
st.pop();
```

## 48. 分割統治とFFTによる条件付き畳み込み: 分割統治FFT
インデックスに順序の制約（$i < j$）がある畳み込み演算を、配列を前半と後半に分割する「分割統治法」と、高速フーリエ変換（FFT）を用いた「通常の畳み込み」を組み合わせて $O(N (\log N)^2)$ で高速に計算する高度な手法。提示された解説では、「文字列の中から等間隔に並んだ特定の文字ペア（$i + k = 2j$ かつ $i < k$）」の総数を数える問題において、条件を畳み込みの形に帰着させてこの手法を適用している。

- 使い方: 配列を中央で分割し、「左半分同士の再帰」「右半分同士の再帰」「左側と右側のFFT（この組み合わせは常に $i < j$ を満たす）」の3パターンの結果を合算する。
- 数学的イメージ: 添字の順序制約 $i < j$ を持つ双線形写像を、直和分解 $V = V_L \oplus V_R$ に基づいて $(V_L \times V_L) \oplus (V_L \times V_R) \oplus (V_R \times V_R)$ の3つの空間の演算へと再帰的に分解するアルゴリズム。
- 関連: 対象を半分に分割して計算量を落とす基礎的な考え方については [二分探索（Binary Search）](#34) を参照。

### 実装例：順序制約付き畳み込みの論理構造（疑似コード）
```cpp
// i < j を満たす A_i * B_j の畳み込み結果を返す再帰関数
vector<long long> solve(const vector<int>& A, const vector<int>& B) {
    int n = A.size();
    if (n <= 1) return {0}; // 基底ケース

    int mid = n / 2;
    // 配列を前半(L)と後半(R)に分割
    vector<int> AL(A.begin(), A.begin() + mid);
    vector<int> AR(A.begin() + mid, A.end());
    vector<int> BL(B.begin(), B.begin() + mid);
    vector<int> BR(B.begin() + mid, B.end());

    // 1. 両方が前半 (i < j < mid) のケースを再帰的に計算
    vector<long long> CL = solve(AL, BL);
    
    // 3. 両方が後半 (mid <= i < j) のケースを再帰的に計算
    vector<long long> CR = solve(AR, BR);
    
    // 2. 左側(i)と右側(j)の組み合わせ (i < mid <= j) は常に i < j を満たすため、
    // 順序を気にせず通常のFFTによる O(N log N) の畳み込みが可能
    vector<long long> C_cross = convolution(AL, BR); 

    // CL, CR, C_cross の結果を、インデックスのズレ(mid)を考慮して足し合わせる
    return merge_results(CL, C_cross, CR, mid);
}
```

## 49. 末尾への効率的な追加と後方アクセス: emplace_back / end()[-k]
`emplace_back` は配列の末尾に要素を直接構築して追加する関数。余分なコピーや移動処理を省略できるため、`push_back` よりも効率的に動作する。`A.end()[-k]` は、配列の終端（最後の要素の1つ後ろ）を指すイテレータからマイナス方向へ相対的にアクセスすることで、配列の後ろから $k$ 番目の要素を $O(1)$ で素早く取得する簡潔な記法。

- 使い方: `A.emplace_back(値)` / `A.end()[-取得したい後ろからの順番]`
- 数学的イメージ: 有限列 $A=(a_0, \dots, a_{n-1})$ に対し、追加操作は長さを $n+1$ に拡張する写像、後方アクセスは終端の添字 $n$ からの平行移動による要素 $a_{n-k}$ の評価。
- 関連: 任意の位置へのアクセスについては [ランダムアクセス](#30) を、イテレータの基礎的な操作については [erase-remove](#3) を参照。

### 実装例：要素の追加と後ろからの取得
```cpp
vector<int> a = {10, 20, 30};

// 末尾に 40 を直接構築（push_backとほぼ同じ働きだが、内部的に無駄がない）
a.emplace_back(40);
// 現在の a: {10, 20, 30, 40}

// end() は「最後の要素のさらに1つ後ろの無効な位置」を指す
// [-1] を指定するとそこから1つ前に戻るため、一番後ろの 40 が取得できる
int last = a.end()[-1]; 

// 後ろから3番目（20）を取得する
int third_from_last = a.end()[-3];

```

## 50. 数列の高速な畳み込み: 高速フーリエ変換（FFT / NTT）
2つの数列（多項式の係数）を掛け合わせる「畳み込み（Convolution）」を計算するアルゴリズム。すべての組み合わせを愚直に計算すると $O(N^2)$ の時間を要するが、数列を波（周波数成分）の表現に変換して掛け合わせた後、元の波形に逆変換することで $O(N \log N)$ で極めて高速に処理できる。競技プログラミングでは、誤差のない有限体上の変換（数論変換：NTT）をライブラリ経由で呼ぶのが標準的である。

- 使い方: `convolution(a, b)` （※AtCoder Libraryなどの提供関数を使用）
- 数学的イメージ: 多項式の係数ベクトル表現を、1の原始 $N$ 乗根を用いた離散フーリエ変換 $\mathcal{F}$ によって点値表現へ同型写像し、要素ごとの積（アダマール積）を計算した後、逆変換 $\mathcal{F}^{-1}$ で元の基底へ引き戻す操作。
- 関連: 畳み込みの計算途中にインデックスの順序制約を組み込む応用技法については [分割統治FFT](#48) を参照。

### 実装例：多項式の乗算（畳み込み）
```cpp
// 競技プログラミングでは AtCoder Library (ACL) などを利用してブラックボックスとして扱う
vector<long long> a = {1, 2, 3}; // 多項式: 1 + 2x + 3x^2
vector<long long> b = {4, 5, 6}; // 多項式: 4 + 5x + 6x^2

// FFT（正確には整数上で誤差が出ない数論変換 NTT）による O(N log N) の畳み込み
// 各次数の係数の和（例: x^2 の係数は 1*6 + 2*5 + 3*4 = 28）が計算される
vector<long long> c = convolution(a, b);
// 結果: {4, 13, 28, 27, 18}
```

## 51. ソート済み配列での要素の存在判定: binary_search
提示されたコードは、配列 `s` の各文字列の先頭3文字を削った部分文字列（`substr(3)`）が、配列 `t` の中に存在するかを数えています。`binary_search` は、事前にソートされた配列の中から特定の要素が存在するか（`true` または `false`）を $O(\log N)$ で極めて高速に判定する関数です。

- 使い方: `binary_search(開始位置, 終了位置, 探したい値)`
- 数学的イメージ: 全順序集合の有限列 $T$ が単調増加に整列されているとき、任意の要素 $x$ について命題 $x \in T$ の真偽を区間縮小法により対数時間で評価する写像。
- 関連: 境界のインデックスそのものを特定する自作の二分探索については [二分探索（Binary Search）](#34) を参照。

### 実装例：先頭3文字を削った文字列の存在判定

```cpp
vector<string> t = {"apple", "banana", "cherry"};

// 必須条件: binary_search はソート済みの配列に対してのみ正しく動作する
sort(t.begin(), t.end());

string s = "123apple";
// substr(3) で先頭3文字を削った "apple" が t に含まれるかを O(log N) で判定
if (binary_search(t.begin(), t.end(), s.substr(3))) {
    // "apple" は t に存在するため、このブロックが実行される
}
```

## 52. 条件を満たす直近の要素の探索: 単調スタック（Monotonic Stack）
スタック内の要素が「常に単調減少（または単調増加）」になるように、条件に合わない要素を削除しながらデータを追加していく手法。ご提示のコードは、配列の左から順番に見ていき、「自分より左にあって、自分より大きい一番近い要素はどこか？」を求めている。

ループが二重（`for` の中に `while`）になっているため一見すると $O(N^2)$ に思えるが、**「各要素はスタックに1回だけ入り（`push`）、最大でも1回しか出ない（`pop`）」**という性質がある。そのため、内側の `while` ループは $N$ 回の処理全体を通しても合計で $N$ 回しか実行されず、全体の計算量はならし（Amortized）で $O(N)$ となる極めて効率的なアルゴリズムである。

- 使い方: 新しい要素を追加する前に、スタックの先頭が条件（今回は「新しい要素より大きい」）を満たすまで `pop` し続ける。
- 数学的イメージ: 要素の有限列において、添字 $j < i$ かつ $A_j > A_i$ を満たす最大の $j$ を求める写像を、不要な要素（将来の判定で決して解にならない要素）の枝刈りによって線形時間で評価するアルゴリズム。
- 関連: スタックの基本的な後入れ先出しの構造については [stack](#47) を参照。

### 実装例：左側にある「自分より大きい直近の要素」を見つける
```cpp
// 番兵（極端に大きな値）を入れておくことで、スタックが空になるのを防ぐ
vector<int> val = { (int)1e9 }; 
vector<int> ind = { -1 };

// a = {3, 1, 4, 2} のような配列を順に処理
for (int i = 0; i < n; i++) {
    // 自分(a[i]) 以下の値は、これ以降の要素にとって「直近の大きな値」にはなり得ないため捨てる
    while (val.back() <= a[i]) {
        val.pop_back();
        ind.pop_back();
    }
    
    // この時点でスタックの一番上(back)に残っているのが、条件を満たす直近の要素
    cout << ind.back() << '\n';
    
    // 自分自身も今後の判定の候補としてスタックに積む
    val.push_back(a[i]);
    ind.push_back(i + 1); // 1始まり(1-indexed)で位置を記録
}
```

## 53. 範囲内の最大値を持つ位置を探す: max_element
指定した範囲（配列など）の中に含まれる要素をすべて調べ、その中で「最も大きい値」を指し示すイテレータ（メモリ上の位置・ポインタのようなもの）を返す関数。戻り値は「値そのもの」ではなく「位置」であるため、提示されたコードのように先頭に `*`（間接参照演算子）をつけて `*max_element(...)` とすることで、その位置に入っている「実際の最大値」を取り出している。
ご提示のコードでは、「各アルファベットの出現回数（`cnt` 配列）」の中から「一番多く出現した文字の回数」を取得するために使われている。

- 使い方: `*max_element(開始位置, 終了位置)` で最大値を取得。最小値の場合は `*min_element(...)` を使う。
- 数学的イメージ: 全順序集合の有限部分列 $A$ において、$\forall x \in A, m \ge x$ を満たす最大元 $m = \max(A)$ の存在する座標（インデックス）を線形探索によって特定する写像。
- 関連: イテレータの概念や範囲指定については [erase-remove](#3) を、位置（イテレータ）同士の引き算によるインデックスの計算については [distance] 等を参照。

### 実装例：最大値とそのインデックス（位置）の取得
```cpp
vector<int> v = {10, 50, 30, 20, 40};

// 最大値の「位置」を取得し、* で「値」に変換する
int max_val = *max_element(v.begin(), v.end()); 
// 結果: 50

// イテレータの性質を利用して、先頭からの距離を計算すれば「何番目か」もわかる
int max_index = distance(v.begin(), max_element(v.begin(), v.end()));
// 結果: 1 (50はインデックス1にあるため)
```

## 54. 安全で便利な固定長配列: std::array
C++11から導入された、要素数が固定された配列を扱うためのコンテナです。従来の「生配列（Raw Array）」の「動作が速く、メモリのオーバーヘッドがない」というメリットを完全に保ちながら、「自身のサイズを知らない」「簡単にコピー代入ができない」といった生配列の弱点を克服し、`std::vector` と同じような便利な機能（`size()` や `begin()` などの関数）を使えるようにしたモダンなデータ構造です。

- 数学的イメージ: 固定長のベクトル空間において、生配列の写像構造を保持したまま、自己の次元（要素数）や境界をメタデータとして内包し、標準的な代数操作（代入やイテレータ演算）を安全に適用できるように拡張された構造体。
- 関連: 古典的な固定長配列については [生配列](#45) を、サイズを後から動的に変更できる配列については [ランダムアクセス（vector）](#30) を参照。

```cpp
// 1. 生配列: 速いがあつかいにくい

// 2. vector: 便利だが、後からサイズ変更できる仕組みを持つためわずかに管理コストがある
```

## 55. 生配列のソートとポインタ演算: P, P + N
結論から言うと、生配列の場合は `P + N` で**まったく問題ありません（大正解です）**。

`std::sort` などの標準ライブラリ関数は、本来「開始位置を示すイテレータ」と「終了位置（最後の要素の1つ後ろ）を示すイテレータ」の2つを受け取るように設計されています。
しかし、C++における「イテレータ（反復者）」とは、実は「ポインタ（メモリアドレス）」の概念をクラスとして高度に拡張したものです。そのため、**生配列が持つ「純粋なポインタ」は、そのままイテレータとして扱うことができる**のです。

- 使い方: `sort(配列名, 配列名 + 要素数);`
- 数学的/メモリ的イメージ: 配列の先頭アドレスを基点とし、型のサイズ（`int` なら4バイトなど）を単位とした $N$ 回の平行移動操作によって得られる「終端＋1」の番地を計算する操作。
- 関連: 動的配列のイテレータを用いたソートについては [sortのカスタム比較](#38) などを参照。

### ポインタとイテレータの対応関係
`vector` などのコンテナと、生配列での記述方法の違いは以下のようになります。

| 役割 | `std::vector<int> V(N);` の場合 | `int P[10];` (要素数N) の場合 |
| :--- | :--- | :--- |
| **先頭** | `V.begin()` | `P` （先頭要素のアドレス） |
| **終端** | `V.end()` | `P + N` （先頭からN個分後ろのアドレス） |

### 実装例：生配列のポインタ演算
```cpp
int N = 5;
int P[10] = {4, 1, 5, 2, 3}; // 残り5つは0で初期化される

// P は「配列の先頭(P[0])のアドレス」
// P + N はそこから N 個進んだ「P[5]のアドレス（ソート対象の直後）」を指す
sort(P, P + N); 

// 結果として、P[0] 〜 P[4] までの範囲が正しく {1, 2, 3, 4, 5} にソートされる
```

## 56. 接尾辞の辞書順ソート: Suffix Array（接尾辞配列）
与えられた文字列や配列の「すべての接尾辞（Suffix：ある位置から末尾までの連続部分列）」を辞書順（または数値の昇順）に並び替え、その「開始インデックス」を配列として返す高度なデータ構造。
愚直に接尾辞を生成して `sort` すると計算量が $O(N^2 \log N)$ かかってしまうが、AtCoder Library (ACL) の `suffix_array` は SA-IS（Suffix Array - Induced Sorting）という非常に強力なアルゴリズムを内部で用いており、これを $O(N)$ または $O(N \log N)$ という魔法のような速度で構築できる。高速な文字列検索や、LCP配列（最長共通接頭辞）と組み合わせた高度な処理の土台として使われる。

- 使い方: `suffix_array(文字列や配列)` （※ `<atcoder/string>` のインクルードが必要）
- 数学的イメージ: 長さ $N$ の列 $A$ に対し、$i$ 番目以降の部分列を $A[i \dots N-1]$ としたとき、全順序集合 $\{ A[0], A[1], \dots, A[N-1] \}$ を辞書順で比較・整列した際の「元の添字 $i$ の列」を生成する写像。
- 関連: ソートの基本については [sortのカスタム比較](#38) などを参照。

### 仕組みのイメージ（文字列 "banana" の場合）
1. すべての接尾辞（インデックス付き）を列挙する
   - `0`: "banana"
   - `1`: "anana"
   - `2`: "nana"
   - `3`: "ana"
   - `4`: "na"
   - `5`: "a"
2. これらを辞書順（アルファベット順）にソートする
   - `5`: "a"
   - `3`: "ana"
   - `1`: "anana"
   - `0`: "banana"
   - `4`: "na"
   - `2`: "nana"
3. このときの「元のインデックスの並び」である `{5, 3, 1, 0, 4, 2}` が Suffix Array となる。

### 実装例：文字列と数値配列の Suffix Array
```cpp
// 1. 文字列に対する Suffix Array
string s = "banana";
vector<int> sa_str = suffix_array(s);
// 結果: {5, 3, 1, 0, 4, 2}

// 2. ユーザーのコードと同じく、数値配列（vector<int>）に対する Suffix Array も可能
// 比較はアルファベットの代わりに数値の大小で行われる
vector<int> p = {10, 20, 10, 30};
vector<int> sa_p = suffix_array(p);
// Suffix一覧:
// 0: {10, 20, 10, 30}
// 1: {20, 10, 30}
// 2: {10, 30}
// 3: {30}
// ソート後: 2({10, 30}), 0({10, 20...}), 1({20...}), 3({30})
// 結果: {2, 0, 1, 3}

// ※ 出力時に x + 1 をしているのは、0始まり(0-indexed)のインデックスを
//   競技プログラミングでよく要求される1始まり(1-indexed)の順位に変換するため。
```

## 57. 複数の値をひとまとめにするデータ構造: tuple (タプル)
C++11から導入された、異なる型の複数の値を1つの変数としてまとめることができる便利な機能です。2つの値をまとめる `std::pair` の拡張版であり、3つ以上の値でも自由に組み合わせることができます。

ご提示のコードにおいて `tuple` を使っている最大の理由は、**「辞書順の比較が自動で定義されているから」**です。

年（Y）、月（M）、日（D）を `tuple<int, int, int>` にまとめて `<=` で比較（`cur <= hei`）すると、「まず年を比較し、年が同じなら月を比較し、月も同じなら日を比較する」という複雑な条件分岐（カレンダーの前後判定）を、たった1行で完璧に処理してくれます。

- 使い方: `tuple<型1, 型2, 型3...> 変数名(値1, 値2, 値3...);`
- 数学的イメージ: 複数の集合の直積（デカルト積） $A \times B \times C \dots$ の要素として、複数の属性（次元）を1つの組（タプル）として扱う概念。辞書式順序（Lexicographical order）が適用される。
- 関連: 2つの値だけを扱う場合は `std::pair` もよく使われます。

### 実装例：tupleの基本的な操作と値の取り出し
```cpp
// 1. エイリアス（別名）をつけて短くする（ご提示のコードのやり方）
using T = tuple<int, int, int>;
T my_date(2019, 4, 30);

// 2. 異なる型を混ぜることも可能
tuple<int, string, double> item(1, "apple", 3.14);

// 3. 値を取り出すときは get<インデックス>(変数) を使う
int id = get<0>(item);         // 1
string name = get<1>(item);    // "apple"

// 4. 辞書順比較の例（日付の比較）
tuple<int, int, int> date1(2019, 4, 30);
tuple<int, int, int> date2(2019, 5, 1);

if (date1 < date2) {
    // 年は同じ（2019）だが、月が 4 < 5 なのでこちらが実行される
    cout << "date1のほうが古い" << endl;
}
```

## 58. 固定長のビット列操作と区間マスク: bitset とビット反転(~)
ご推察の通り、`std::bitset` は数値を「0と1の並び（2進数）」として直接扱うための機能です。通常の整数（`int` など）のサイズ制限（32ビットや64ビット）を超えて、100ビットなどの長大なビット列を扱うことができ、複数のフラグ（状態）や集合を管理するのに適しています。

`~` は「ビット反転（NOT）」を行う演算子です。0を1に、1を0にひっくり返します。
ご提示のコード `~(~o << (r1 - l1)) << l1` は、これを利用して「$l_1$ 番目から $r_1 - 1$ 番目までが 1、それ以外が 0」という区間を表すビットマスクを生成する非常に鮮やかなテクニックです。

- 使い方: `bitset<ビット数> 変数名;` / `~変数` (反転) / `変数 << シフト数` (左シフト)
- 数学的イメージ: 全体集合 $U = \{0, 1, \dots, N-1\}$ の部分集合 $A$ を、指示関数 $\chi_A(x)$ を用いた長さ $N$ のビットベクトルとして表現し、集合の積和演算（和集合、積集合など）をプロセッサの論理演算命令によって並列計算する手法。

### 実装例：区間マスク生成のステップ分解
例として、$l_1 = 2, r_1 = 5$（区間の長さ3）のマスクを作る過程を右側（最下位ビット）から順に追ってみましょう。

```cpp
bitset<8> o; // 初期状態: 00000000

// 1. ~o で全体を反転させ、全てを1にする
// 11111111

// 2. 長さ(5 - 2 = 3)だけ左にシフトする (空いた右端には0が入る)
// ~o << 3  =>  11111000

// 3. 全体をもう一度 ~ で反転させる（これで「長さ3の1の塊」が右端にできる）
// ~(~o << 3) => 00000111

// 4. 指定された開始位置 l1 (2) まで左シフトして完成！
// 00011100
```

## 59. 複雑な文字列のルールを記述する魔法の言語: 正規表現 (Regular Expression)
正規表現（Regex）は、「特定の文字が含まれているか」といった単純な検索ではなく、「最初の文字は特定の4種類のどれかで、次の文字は特定の13種類のどれか」といった**複雑な文字列のルール（パターン）を、短い記号の組み合わせだけで表現する**ための特別な手法（ミニ言語）です。非常に特別で、そして強力です。

ご提示のコード `[HDCS][A2-9TJQK]` は、トランプのカードが正しいフォーマットか判定する非常に賢い書き方です。もしこれを正規表現を使わずに普通の `if` 文で書こうとすると、「1文字目が H か D か C か S のどれかで、かつ、2文字目が A か 2 か 3 か...」と途方もなく長い条件分岐が必要になってしまいます。

- 使い方: `re.compile(r'パターン')` でルールをコンパイル（準備）し、`.match(文字列)` でそのルールに完全に一致するかを判定する。
- 数学的イメージ: アルファベット $\Sigma$ 上の文字列の集合（正規言語）を、和集合（`[]` や `|`）、連接、およびクリーネ閉包（`*`）の演算によって有限の長さの数式として定義するシステム。
- 関連: 文字列の一部を切り出して調べる基本的な操作については [substr](#51) を参照。

## 正規表現の構成要素: メタ文字とパターン記述
文字列の探索や置換を行うため、特定の文字集合、反復回数、テキスト内の位置などを抽象化して指定する特殊記号群（メタ文字）。主要な記号は以下の4カテゴリに分類される。

**1. 文字クラス（文字の種類を指定）**
- `.` : 任意の1文字（改行を除く）
- `\d` / `\D` : 数字 (`[0-9]`) / 数字以外 (`[^0-9]`)
- `\w` / `\W` : 英数字と `_` (`[a-zA-Z0-9_]`) / それ以外
- `\s` : 空白文字（スペース、タブ、改行など）
- `[...]` / `[^...]` : カッコ内の任意の1文字 / 含まれない1文字

**2. 量指定子（回数や連続を指定）**
- `*` / `+` / `?` : 0回以上 / 1回以上 / 0回または1回
- `{n}` / `{n,}` / `{n,m}` : ちょうどn回 / n回以上 / n回以上m回以下

**3. アンカー（位置を指定）**
- `^` / `$` : 行の先頭 / 行の末尾
- `\b` : 単語の境界（単語の始まりや終わり）

**4. グループ化と選択**
- `(...)` : グループ化（複数の文字をまとめる、キャプチャする）
- `|` : OR論理和（例：`cat|dog` は "cat" または "dog"）

- 数学的イメージ: アルファベット $\Sigma$ 上の文字列の集合において、和集合 `|`、連結、およびクリーネ閉包 `*` の正規演算を用いて、特定の部分集合（正規言語）を代数的に構成する表現形式。

### 実装例：メタ文字を組み合わせたマッチングと探索
```cpp
// C++で正規表現を書く際は、バックスラッシュ(\)のエスケープを省略できる
// Raw文字列リテラル R"(...)" を使うのが一般的。

// 例1: ^ と $ を使い、文字列全体が「数字3桁-数字4桁」であるかを完全一致判定
regex zip_pattern(R"(^\d{3}-\d{4}$)");
bool is_valid = regex_match("123-4567", zip_pattern); // true

// 例2: \b と (...) と | を組み合わせた文中からの部分探索
string text = "My pets are a cat and a dog.";
regex pet_pattern(R"(\b(cat|dog)\b)"); // 独立した単語としての cat か dog

smatch match;
if (regex_search(text, match, pet_pattern)) {
    // 最初にマッチした "cat" が match.str() に格納される
}
// 関連: [正規表現による置換と非重複マッチ](#88-正規表現による置換と非重複マッチ-regex_replace)
```
---

### 実装例：パターンの分解
ご提示の正規表現 `r'[HDCS][A2-9TJQK]'` は、角括弧 `[]` で囲まれた2つのブロック（文字クラスと呼びます）が繋がってできています。

```python

# -> 'H', 'D', 'C', 'S' のいずれか「1文字」にマッチする（スートを表す）
```

## 60. 重複を許す自動ソート集合: multiset
通常の `std::set` は「重複を許さない（同じ値を入れても1つにまとめられる）」データ構造ですが、`std::multiset` は「重複を許す」集合です。追加したデータは自動的に辞書順や昇順にソートされ、要素の検索や追加・削除が $O(\log N)$ で高速に行えます。

### 実装例：multisetによる重複カウント
```cpp
multiset<string> mset;
mset.insert("apple");

// "apple" は2回入れたので 2 が返る
```

## 62. 全体一致か部分一致か: regex_match と regex_search

## 63. 動的配列のサイズを後から変更する: resize
`std::vector::resize(N)` は、動的配列（`vector`）の要素数を、後から指定した数 `N` に強制的に変更する関数です。
元のサイズより大きくした場合は、増えた分の「箱」が新しく作られ（数値型なら自動的に `0` で初期化されます）、元のサイズより小さくした場合はみ出した後ろの要素は削除されます。

ご提示のコードでは、行ごとにデータ数 `l` が異なる「ジャグ配列（ギザギザな2次元配列）」を作るために使われています。あらかじめ `resize(l)` で `l` 個の箱を用意しておくことで、その後のループで `push_back` を使わずに直接 `a[i][j]` とインデックスを指定して安全にデータを書き込むことができます。

- 使い方: `変数名.resize(新しい要素数);`
- 数学的イメージ: 長さ $k$ の可変長ベクトルを、指定した次元 $N$ の空間へ拡張（または縮小）する操作。拡張部分はデフォルト値（ゼロ元）で埋められる。
- 関連: 箱を事前に用意せず1つずつ末尾に追加していく書き方については [emplace_back](#49) を参照。

### 実装例：resize の働きと push_back との違い
```cpp
vector<int> v = {10, 20}; // 現在のサイズは 2

// 現在の v: {10, 20, 0, 0, 0} (新しく増えた3つの箱には 0 が入る)

// 既に箱が用意されているので、直接インデックスでアクセス・代入できる
v[2] = 30; 
// 現在の v: {10, 20, 30, 0, 0}

// サイズを 2 に減らす
v.resize(2);
// 現在の v: {10, 20} (後ろにはみ出た要素は切り捨てられて消える)
```

## 64. 最大公約数とユークリッドの互除法: GCD
ご推察の通りです！（※おそらく「GCD」のタイポですね）。
`GCD` は **G**reatest **C**ommon **D**ivisor の略で、2つの数の「最大公約数」を求める関数として、競技プログラミングなどで非常によく使われる名前です。

このたった数行の関数の中には、「**ユークリッドの互除法**」という、紀元前から存在する非常に強力で高速な計算手法が詰まっています。

- 使い方: `GCD(値1, 値2)`
- 数学的イメージ: 2つの自然数 $a, b$ の最大公約数を $\gcd(a, b)$ としたとき、$\gcd(a, b) = \gcd(b, a \bmod b)$ が成り立つという定理を利用し、剰余が $0$ になるまで再帰的に適用して最大公約数を対数時間 $O(\log(\min(A, B)))$ で導出するアルゴリズム。
- 関連: 最大公約数を利用して「最小公倍数（LCM）」を求める処理が、このコードの `main` 関数で行われています。

### 実装の解説

#### ユークリッドの互除法 (`GCD` 関数)
```cpp
long long GCD(long long x, long long y) {
    // y が 0 でないなら GCD(y, xをyで割った余り) を再帰的に呼び出す
    // y が 0 になったら、その時点の x が最大公約数なので返す
    return (y ? GCD(y, x%y) : x);
}
```

## 65. 集合内の要素の有無（0 か 1 か）を判定する: set::count

- 数学的イメージ: 集合 $S$ と要素 $x$ について、指示関数 $\chi_S(x)$ （$x \in S$ なら $1$、$x \notin S$ なら $0$）を $O(\log N)$ で評価する写像。
- 関連: 重複を許す集合でのカウントについては [multiset](#60) を参照。

### 実装例：setのcountとモダンな代替手段

```cpp
if (s.count(10)) {
    cout << "10は存在します" << endl; // こちらが実行される
}

// （補足）C++20以降は、より意味が直感的な contains 関数も追加されました
if (s.contains(10)) {
```

## 66. 2点間の距離（直角三角形の斜辺）を安全に求める: hypot
`hypot` は「Hypotenuse（直角三角形の斜辺）」の略です。数学的には、与えられた $x$ と $y$ に対して $\sqrt{x^2 + y^2}$ を計算して返します。
ご提示のコードのように、`hypot(x - crrx, y - crry)` とすることで、「現在の点 $(crrx, crry)$ から次の点 $(x, y)$ までの直線距離」を求めることができます。

なぜ自分で `sqrt(x*x + y*y)` と書かずに `hypot` を使うのか？最大の理由は**「オーバーフロー（桁あふれ）を防ぐ安全性」**です。
例えば $x$ や $y$ が非常に大きい値の場合、自分で `x * x` を計算した瞬間に型の限界を超えてバグる危険がありますが、`hypot` は内部で賢く計算し、途中の桁あふれを防いで正確な距離を算出してくれます。

- 使い方: `hypot(xの差分, yの差分)` （※ `<math.h>` または `<cmath>` が必要）
- 数学的イメージ: 2次元ユークリッド空間上の2点間のノルム（距離）を、ピタゴラスの定理を用いて評価する写像。
- 関連: 平方根を求める基本関数については `sqrt` を参照。

### コードの全体的な動き（何をやっているか）
このプログラムは、**「原点 $(0,0)$ から出発し、$N$ 個の座標を順番に巡って、最後にまた原点に戻ってくるまでの総移動距離」**を計算しています。

1. `ans = 0` （移動距離の合計）、`crrx = 0, crry = 0` （現在の座標：最初は原点）
2. ループ内で次の目的地 $(x, y)$ を受け取る
3. `ans += hypot(...)` で、現在地から目的地までの距離を計算して `ans` に足す
4. `crrx = x, crry = y` で、現在地を更新する
5. ループを抜けたら、最後の場所から原点 $(0,0)$ に戻る距離を `ans` に足す
6. `printf` で小数点以下10桁まで出力する

## 67. 複雑な条件を1行で判定する正規表現の解読
ご提示の正規表現 `^[A-Z][1-9][0-9]{5}[A-Z]$` は、特定のIDやパスワードのような、**「決まった8文字の形式」**に完全に一致するかどうかを判定しています。

記号を一つずつ分解して翻訳していくと、以下のようになります。

*   **`^`** : 文字列の「先頭」であることを表す記号
*   **`[A-Z]`** : 英大文字（A〜Z）のどれか **1文字**
*   **`[1-9]`** : 1〜9の数字のどれか **1文字**（※ `0` は許容されない）
*   **`[0-9]{5}`** : 0〜9の数字が **ちょうど5文字** 続く（`{5}` は直前のルールの5回繰り返し）
*   **`[A-Z]`** : 英大文字（A〜Z）のどれか **1文字**
*   **`$`** : 文字列の「末尾」であることを表す記号

### 📝 つまり、どんな文字列が「Yes」になる？
これらをすべて繋げると、以下の条件を**すべて**満たす文字列だけが「Yes」になります。

1.  **全体でちょうど8文字**であること（先頭 `^` から末尾 `$` まで余計な文字がないため）
2.  **1文字目** と **8文字目** が英大文字であること
3.  **真ん中の6文字** が数字であること
4.  ただし、**数字の最初の1文字目は `0` ではない**こと（＝真ん中の数字は `100000` から `999999` の範囲であること）

**✅ Yesになる例:**
*   `A123456Z`
*   `X900000Y`

**❌ Noになる例:**
*   `a123456Z` （最初が小文字）
*   `A012345Z` （数字の先頭が0）
*   `A12345Z` （数字が5桁しかない）
*   `A1234567Z` （数字が7桁ある）

> **補足:** Kotlinの `.matches(S)` は、そもそも「文字列全体が一致するか」を判定する関数なので、実は先頭の `^` と末尾の `$` は省略しても同じ動作をします。しかし、他の言語やツールでも使える汎用的な書き方として、`^` と `$` を明示的につけるのは非常に良い習慣です。

`std::ranges::contains` は、C++23から導入された非常に直感的で便利な関数です。名前の通り、「ある範囲（文字列や配列など）の中に、探したい要素が含まれているか（containされているか）」を調べ、`true` か `false` を返します。

## 69. 正規表現の `*` を使った部分集合（包含）判定トリック
はい、ご推察の通りです！このコードはまさに**「入力された文字列 `s` のすべての文字が、元の文字列 `S`（から作った `r[0]`）に含まれているか」**を判定しています。

前回の `ranges::contains` を使ったコード（番号68）とやっている目的は全く同じですが、こちらは正規表現の性質を逆手に取った、非常にユニークでトリッキーな手法です。

- 使い方: 文字列の各文字の後ろに `*` を付けた正規表現を作り、**ソート済みの**検索対象文字列とマッチさせる。
- 数学的イメージ: アルファベットの順序集合 $S = \{c_1, c_2, \dots, c_k\}$ （ただし $c_1 < c_2 < \dots < c_k$）に対して、正規言語 $L = c_1^* c_2^* \dots c_k^*$ を定義し、任意の文字列 $w$ を昇順ソートした $w'$ が $w' \in L$ を満たすか調べることで、$w$ が $S$ の部分集合のみから構成されているかを判定する手法。

### なぜこれで「全て含まれているか」が判定できるのか？
正規表現において、**`*`（アスタリスク）は「直前の文字が0回以上繰り返される」**という意味を持ちます。

1. **正規表現を作る (`x[i] += c; x[i] += "*";`)**
   もし元の文字列 `S` が `"abc"` だった場合、このループを通ると `x[0]` は **`a*b*c*`** という正規表現になります。
   これは「`a` が0回以上続き、その後に `b` が0回以上続き、その後に `c` が0回以上続く文字列」にマッチします。

2. **検索対象をソートする (`sort(s.begin(), s.end());`)**
   判定したい単語 `s` が `"bca"` だったとします。これをそのまま `a*b*c*` と当てはめても順番が違うのでハジかれます。しかし、アルファベット順にソートすると **`"abc"`** になります。

3. **マッチ判定 (`regex_match(s, r[0])`)**
   ソートされた `"abc"` を `a*b*c*` に当てはめると、見事にマッチします（`a`が1回、`b`が1回、`c`が1回）。
   もし `s` が `"bbaa"` だった場合、ソートして `"aabb"` となります。これも `a*b*c*`（`a`が2回、`b`が2回、`c`が0回）にマッチします。
   逆に、`S` に含まれない `"z"` が入っていた場合、ソート結果に `z` が混ざるため `a*b*c*` には絶対にマッチしません。

### ⚠️ このコードの隠れた罠（注意点）
この美しいトリックが完璧に機能するためには、**「大元の文字列 `S` と `T` 自体も、あらかじめアルファベット順にソートされていて、かつ重複がない（"abc"のような状態）」という大前提が必要です。**

もし問題の入力として `S` が `"cba"` のような順番で与えられた場合、生成される正規表現は `c*b*a*` になります。これに対して、クエリ文字列 `"bca"` をソートした `"abc"` をマッチさせようとすると、「`c`の後に`a`が来る」という順番に反するため、**本来は含まれているはずなのに False になってしまう**というバグを抱えています。
（※問題の仕様で「SとTは昇順で与えられる」と決まっているなら問題ありません！）

## 70.コードの全体像と transform_reduce の解説

全体としてこのプログラムは、**「文字列の中に特定のアルファベットが含まれているか」という探索を、超高速な「ビット（0と1）の計算」に置き換えて判定する**という非常に高度でスマートな手法をとっています。

処理の流れは以下の3ステップです。
1. 高橋語(S)と青木語(T)の文字列を、それぞれ「どのアルファベットが使われているか」を示す **1つの整数（ビットマスク）に圧縮** する。
2. ビット演算を利用して、「高橋語にしかない文字」と「青木語にしかない文字」の集合を作る。
3. 質問される単語 `w` も整数に圧縮し、2で作った集合と重なる部分があるかで一瞬で判定する。

この「文字列を1つの整数に圧縮する」という最も重要な仕事をしているのが `transform_reduce` です。

## transform_reduce とは何をしているのか？

`std::transform_reduce` はC++17で追加された関数で、名前の通り **「要素を変換（transform）」** してから **「1つに集約（reduce）」** する作業を同時に行います。

該当のコードを分解して解説します。
```cpp
transform_reduce(
    begin(S), end(S),           // ① 処理する範囲（文字列Sの最初から最後まで）
    0U,                         // ② 集約の初期値（符号なし整数の0）
    bit_or{},                   // ③ どうやって集約するか（ビットごとの論理和 '|' を使う）
    [](const char c){ return 1U << (c - 'a'); } // ④ どう変換するか
);
```

【ステップA】変換 (Transform) の動き
④のラムダ式（無名関数）が、文字列の1文字1文字に対して実行されます。
c - 'a' は、文字をアルファベット順の数字（a=0, b=1, c=2...）に変換します。
1U << 数字 は、1 をその数字の数だけ左にズラす（シフトする）操作です。

文字が 'a' なら、0回ズラす → 0000...0001 (10進数の1)

文字が 'b' なら、1回ズラす → 0000...0010 (10進数の2)

文字が 'c' なら、2回ズラす → 0000...0100 (10進数の4)

これで、各アルファベットが「自分専用のフラグ（1つのビット）」に変換されます。

【ステップB】集約 (Reduce) の動き
②の初期値 0 をベースに、ステップAで変換されたビットたちを、③の bit_or（両方のビットを足し合わせるような計算）で全て合体させます。

例えば文字列が "abc" だった場合：
0000001 (a) | 0000010 (b) | 0000100 (c) = 0000111

結果として、どんなに長い文字列でも、「aが含まれていれば右から1番目が1、bなら2番目が1...」という情報を記録した【たった1つの整数】に圧縮されるのです。

全体的な判定ロジック（ビット演算による集合操作）
圧縮が終わると、文字列の比較は「整数のビット計算」という、コンピュータが最も得意な超高速処理に変わります。

```cpp
// 高橋語(S)に「しか」ない文字の集合を作る
// T_mask を ~(反転) して0と1をひっくり返し、S_mask と &(論理積: 両方1なら1) を取る
// → 数学の「差集合 (S - T)」を計算しているのと同じ！
const unsigned only_takahashi = S_mask & ~T_mask;

// 青木語(T)に「しか」ない文字の集合 (T - S)
const unsigned only_aoki = ~S_mask & T_mask;
```
最後に、入力された単語 w も同じように w_mask という整数に圧縮し、以下の判定を行います。

```cpp
if (w_mask & only_takahashi)
```
これは「単語 w のビットと、高橋語特有のビットで、同じ場所が1になっている（＝特有の文字が含まれている）箇所があるか？」をチェックしています。もし1つでも重なっていれば、それは間違いなく高橋語だと判定できる、という美しいロジックです。

## 71. 状態のビット圧縮(Bitmask)と超高速ペア判定
「o（解ける）」「x（解けない）」という複数問の正解状況を、1つの「2進数の整数」として丸ごと記憶（圧縮）し、ペアの判定をビット演算を用いて爆速で行う競技プログラミングの最強テクニックの1つです。

通常なら「2人のペアを選ぶ」×「M問すべてチェックする」で $O(N^2 \times M)$ の時間がかかるところを、M問のチェックを「CPUの1回の計算（論理和）」に置き換えることで、$O(N^2)$ へと劇的に高速化しています。

- 使い方: `p[i] |= (1 << j)` (特定のビットを立てる) / `p[i] | p[j]` (状態の合体)
- 数学的イメージ: 各参加者の能力をブール代数 $\{0, 1\}^M$ の元として扱い、2つの元の論理和（和集合）が全体集合 $11\dots1$ と等しくなるかを調べる操作。

### 実装例：ABC282 B "Let's Get a Perfect Score" 最適解
```cpp
#include <iostream>
#include <vector>
#include <string>

using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    
    vector<int> p(n, 0); // 参加者の能力を整数（ビットマスク）で保存
    for (int i = 0; i < n; i++) {
        string s;
        cin >> s;
        // 1. 文字列からビットマスクへの変換
        for (int j = 0; j < m; j++) {
            if (s[j] == 'o') {
                // j番目のビットを立てる（例: j=2なら 00100 を足し合わせる）
                p[i] |= (1 << j); 
            }
        }
    }
    
    int ans = 0;
    // 2. 「全問正解」の状態を作る
    // 1をM回左シフトして1を引くと、右からM個のビットが全て1になる
    // 例: M=3なら (1000) - 1 = 0111 (10進数で7)
    int perfect = (1 << m) - 1; 
    
    // 3. ペアの全探索
    for (int i = 0; i < n; i++) {
        for (int j = i + 1; j < n; j++) {
            // 2人の能力を合体(OR)させて、perfectと完全に一致するか一発で判定！
            if ((p[i] | p[j]) == perfect) {
                ans++;
            }
        }
    }
    cout << ans << endl;
}
```

## 72. 名前のない使い捨て関数とカスタムソート: ラムダ式 (Lambda)
ラムダ式は、わざわざ `main` 関数の外で `int calc() { ... }` のように名前をつけて関数を定義しなくても、**使いたいその場所で直接関数の中身を書き込める** C++11 からの強力な機能です。

ご提示のコードでは、`std::sort` の「並び替えのルール」を自分好みに改造するために使われています。

### 1. ラムダ式の基本構成
ラムダ式は、基本的に以下の3つのパーツ `[]` `()` `{}` で構成されます。

```cpp
auto 変数名 = [キャプチャ](引数) { 
    処理内容 
};
```
[] (キャプチャ): ラムダ式の「外側」にある変数を使いたい時に指定します。今回は外の変数を使わないので空っぽ [] です。（※前回の ranges::contains の解説であった [&S] は、「外にある変数 S を参照で使うよ」という意味です）

() (引数): 普通の関数と同じく、受け取るデータを書きます。

{} (処理内容): 普通の関数と同じく、実行するコードを書きます。

### 2. 今回のラムダ式がやっていること（並び替えルールの自作）

std::sort は通常、数字を「小さい順（昇順）」に並び替えます。しかし、今回の ABC323-B の問題では以下の複雑なルールで並び替える必要があります。

勝った数（a.first）が多い順（降順）

もし勝った数が同じなら、参加者番号（a.second）が小さい順（昇順）

これを std::sort に教えるために、「要素 a と b を比べたとき、a を前に置くべきなら true を、そうでないなら false を返す関数」を作ります。
```cpp
// win[i] は {勝った数, 参加者番号} のペア
auto cmp = [](pair<int, int> a, pair<int, int> b) {
    // もし勝った数が違うなら、勝った数が多い方を前にする（a > b なら true）
    if(a.first != b.first) return a.first > b.first;
    
    // 勝った数が同じなら、参加者番号が小さい方を前にする（a < b なら true）
    return a.second < b.second;
};

// 作ったルール(cmp)を sort に渡す！
sort(win.begin(), win.end(), cmp);
```
このように、sort の第3引数にラムダ式を渡すことで、「独自の優先順位を持ったソート」をたった数行で実現しています。競技プログラミングでは必須級のテクニックです。

## 73. 外の変数にアクセスする: 参照キャプチャ `[&]`
前回のラムダ式は `[]`（キャプチャなし）でしたが、今回は **`[&]`（参照キャプチャ）** が使われています。

ラムダ式は通常、自分が定義された場所の「外側」にある変数（今回の場合は `win`）を勝手に使うことができません。そこで `[&]` と書くことで、**「このラムダ式の外側にある変数を、そのまま（参照として）自由に使っていいよ」**という許可を与えています。

### コードの動き（インデックスソート）
今回のコードは、ペアの配列を作らずに、**「参加者番号（インデックス）だけが入った配列 `p`」** を並び替えています。

1. `iota(p.begin(), p.end(), 0);`
   配列 `p` を `0, 1, 2, ... n-1` という連番で埋める便利な関数です。（これで参加者番号のリストができます）
2. `auto cmp = [&](int i, int j) { return win[i] > win[j]; };`
   「番号 `i` と番号 `j` を比べたとき、**外側にある `win` 配列**を見て、勝った数が多い方を前にする」というルールを作っています。

## 74. 同点の順序を保つ魔法: stable_sort
ここがこのコードの最も賢いところです。
前回のラムダ式では、「勝った数が同じなら、番号が小さい順にする」というルールをわざわざ `if` 文で書いていました。しかし今回のラムダ式には `win[i] > win[j]` しか書かれていません。

なぜこれで「勝った数が同じなら、番号が小さい順」になるのか？
その秘密が **`std::stable_sort`（安定ソート）** です。

通常の `std::sort` は処理が爆速な代わりに「不安定（Unstable）」であり、同点のデータがあった場合、それらが元の順序のまま並ぶ保証がありません（順番がシャッフルされてしまうことがあります）。
一方、`std::stable_sort` は、**「同点のデータがあった場合、ソートする前の元の順番を絶対に崩さない」** という性質を持っています。

配列 `p` には最初 `0, 1, 2...` と番号が小さい順に入っていたため、勝った数が同点であれば、`stable_sort` が自動的に「最初に入っていた順（＝番号が小さい順）」を保ってくれるのです。

- 使い方: `stable_sort(始まり, 終わり, ルール)`
- 関連: 順序の保持が必要ない（または自分でルールを完全に記述する）場合は通常の `sort` を参照。

## 75. 座標から正しい角度（偏角）を安全に求める: atan2
`atan2(y, x)` は、原点 $(0,0)$ から指定した座標 $(x, y)$ への直線が、X軸のプラス方向となす角度 $\theta$（ラジアン）を求めるための関数です。

なぜ普通の「アークタンジェント `atan(y/x)`」を自分で計算してはいけないのでしょうか？それには2つの大きな理由（バグの元）があります。

1. **ゼロ除算エラーの回避:** $x = 0$ のとき `y/x` を計算しようとするとプログラムがクラッシュしてしまいますが、`atan2` は内部で安全に処理し、正しく $\pi/2$（90度）や $-\pi/2$（-90度）を返してくれます。
2. **正しい象限（向き）の判定:** 例えば座標 $(1, 1)$ と $(-1, -1)$ は、どちらも $y/x = 1$ になってしまい、普通の `atan` では「右上」なのか「左下」なのか区別がつきません。`atan2` は $x$ と $y$ 両方の符号を独立して確認するため、全方位（$-\pi$ から $\pi$ の間）で正しい角度を返してくれます。

- 使い方: `atan2(y座標, x座標)` （※引数の順番が数学の慣習とは逆で **y, x** になる言語がほとんどなので要注意です！）
- 数学的イメージ: 2次元平面上の点 $(x, y)$ の極座標表示 $(r, \theta)$ における偏角 $\theta = \arg(x + iy)$ を正しく評価する関数。
- 関連: 斜辺（距離 $r$）を安全に求める関数については [hypot](#66) を参照。

### コードの全体的な動き（何をやっているか）
ご提示のコードは、入力された座標 $(x, y)$ の「方向」を保ったまま、**「原点からの距離がちょうど 1 になる座標（単位ベクトル）」**に変換して出力しています。

1. `theta = atan2(y, x)` で、その座標がある方角 $\theta$ を取得します。
2. `cos(theta)` と `sin(theta)` を出力します。三角関数の性質上、これにより「その方角 $\theta$ にある、半径1の円周上の座標」が求まります。

（※これは、以前紹介した関数を使って `x / hypot(x, y)` と `y / hypot(x, y)` を出力するのと同じ結果になります。）

## 76. ベクトルから集合を生成: イテレータ範囲コンストラクタ

## 81. Monge行列の性質: 局所条件と大域条件の同値性
Monge行列の大域的な定義（任意の四角形に対する不等式）が、隣接する $2 \times 2$ 部分行列の局所的な条件と同値であることの証明と、それを活用した高速な判定手法。

- 概要: 局所的な条件を満たすことさえ示せば大域的にもMonge性が保証されるため、判定やDPの最適化条件の確認において、計算量を $O(H^2W^2)$ から $O(HW)$ に劇的に落とすことができる。
- 数学的イメージ: 局所条件の差分 $A_{i+1,j+1} - A_{i+1,j} \le A_{i,j+1} - A_{i,j}$ を縦に連鎖させ、横方向に総和（望遠鏡和）をとって中間項を相殺することで大域条件が導出される。

### 実装例：同値性の証明と O(HW) での判定ロジック
【同値性の証明】
大域条件: 任意の $i < k, j < l$ について $A_{i,j} + A_{k,l} <= A_{k,j} + A_{i,l}$
局所条件: 任意の $i, j$ について $A_{i,j} + A_{i+1,j+1} <= A_{i+1,j} + A_{i,j+1}$

1. 大域条件 => 局所条件 (必要性)
   大域条件の式において、$k = i+1, l = j+1$ を代入すれば局所条件と完全に一致するため自明。

2. 局所条件 => 大域条件 (十分性)
   局所条件を移項し、行間の差分の形にする。
   $A_{i+1,j+1} - A_{i+1,j} <= A_{i,j+1} - A_{i,j}$
   
   この関係を i から k-1 まで縦方向に連鎖させると、任意の列 c について以下が成り立つ。
   $A_{k,c+1} - A_{k,c} <= A_{i,c+1} - A_{i,c}$
   
   次に、これを列 c = j から l-1 まで横方向に足し合わせる（総和をとる）。
   $Σ (A_{k,c+1} - A_{k,c}) <= Σ (A_{i,c+1} - A_{i,c})$
   
   両辺ともに「望遠鏡和（Telescoping sum）」となり中間項が全て打ち消し合うため、両端のみが残る。
   $A_{k,l} - A_{k,j} <= A_{i,l} - A_{i,j}$
   
   最後にこれを移項すれば、大域条件 $A_{i,j} + A_{k,l} <= A_{k,j} + A_{i,l}$ が導出され証明完了。

```cpp
// 上記の定理により、Monge行列の判定は隣接2x2のみ（局所条件）を見れば十分となる
bool is_monge(const vector<vector<int>>& A) {
    int H = A.size(), W = A[0].size();
    for (int i = 0; i < H - 1; ++i) {
        for (int j = 0; j < W - 1; ++j) {
            // 局所条件を1つでも破っていればMongeではない
            if (A[i][j] + A[i+1][j+1] > A[i+1][j] + A[i][j+1]) {
                return false;
            }
        }
    }
    return true;
}
// 関連: [Monge DPによる計算量削減](#monge-dpによる計算量削減)
```

## 83. 2の累乗判定と高速な素因数除去: has_single_bit とダブリング
値が2の累乗であるかを判定する `std::has_single_bit` （C++20）と、ダブリング（繰り返し二乗法）の考え方を応用して特定の素因数（ここでは3）を高速に割り切るアルゴリズム。

- 使い方: bool b = has_single_bit(N);
- 数学的イメージ: 整数 $N$ が $N = 2^a \cdot 3^b$ の形で表現できるか判定するため、数列 $S_i = 3^{2^i}$ を用いて $O(\log b)$ で $3^b$ の成分を完全に除算し、残余 $N'$ が $N' \in \{2^k \mid k \ge 0\} \iff \text{popcount}(N') = 1$ を満たすか確認する操作。

### 実装例：N が 2^a * 3^b の形で表せるか判定
```cpp
unsigned long N = 144; // 144 = 2^4 * 3^2

// 1. ダブリングによる高速除算: 3, 9, 81... と二乗しながら割れる最大サイズを探索
vector<unsigned long> pow_3{3};
for (int i = 0; i < 5 && N % pow_3[i] == 0; ++i) {
    pow_3.emplace_back(pow_3[i] * pow_3[i]);
}

// 見つけた大きな塊から順に割っていく（O(log b)で3の素因数を全て除去）
// 関連: [コンテナの空判定](#2-コンテナの空判定-stdempty-非メンバ関数)
while (!empty(pow_3)) {
    if (N % pow_3.back() == 0) {
        N /= pow_3.back();
    }
    pow_3.pop_back(); // 関連: [末尾要素の取得と一様初期化](#3-末尾要素の取得と一様初期化-back-と-)
}

// 2. 残った N が2の累乗（2進数表現で '1' が1つだけ）か判定
bool is_valid = has_single_bit(N); // 3を割り切った残余は N=16 (10000_2) なので true
```

## 87. ダブリング除算の時間計算量: O(log log N)
先ほどのアルゴリズムで特定の素因数を除去する処理が、$O(\log N)$ ではなく $O(\log \log N)$ となる数学的背景とその驚異的な速さの理由。

- 概要: 除数を $3^1, 3^2, 3^3 \dots$ と増やすのではなく、$3^1, 3^2, 3^4, 3^8 \dots$ と「指数の指数」で増大させることで、探索空間を二重に対数的に縮小する手法。
- 数学的イメージ: 最大の除数を $k^{2^i}$ としたとき、$k^{2^i} \le N$ の両辺の対数を2回とることで得られる不等式 $i \le \log_2(\log_k N)$ により、ステップ数が合成関数 $\log(\log N)$ で上界付けられる。

### 実装例：計算量が事実上の定数 O(1) になる理由の証明

```cpp
// 最大でも「約 40 回」のループが必要になる。

// 2. 今回のダブリング除算 O(log log N) の場合
// pow_3 配列には 3^{2^i} が入るため、調べる数は以下の通り急速に増大する。
// i=0: 3^1 = 3
// i=1: 3^2 = 9
// i=2: 3^4 = 81
// i=3: 3^8 = 6561
// i=4: 3^16 = 43046721
// i=5: 3^32 = 1853020188851841
// i=6: 3^64 > N (ここで終了)

// 結論: 64bitの限界である超巨大な N を入力しても、ループ回数 i は
// log2(log3(N)) ≒ log2(40) ≒ 5.3 となり、「最大でたったの 6 回」しか回らない。
// 実用上は計算量が全く増えないため、O(1) と同義の圧倒的な速度が出る。
// 関連: [2の累乗判定と高速な素因数除去](#83-2の累乗判定と高速な素因数除去-has_single_bit-とダブリング)
```

## 89. 最大値の更新関数: chmax
競技プログラミングにおいて、現在の変数の値と新しい値を比較し、新しい値が大きければ上書き（更新）する定番のテンプレート関数。C++の標準機能ではなく、競技者がテンプレートとして自作して使うことが多い。

- 使い方: chmax(ans, 比較する値);
- 数学的イメージ: 全順序集合において、現在の状態 $a$ と新たな評価値 $b$ に対して上限をとる代入操作 $a \leftarrow \max(a, b)$ を適用し、値を単調増加させる写像。

### 実装例：複数の候補からの最大値探索
```cpp
// 1. 事前に定義しておくテンプレート関数（参照渡し '&' により直接書き換える）
template<class T> inline bool chmax(T& a, T b) {
    if (a < b) {
        a = b;
        return true; // 更新されたら true を返す（DPの経路復元などで便利）
    }
    return false;
}

// 2. 実際の使用例
int a = 1, b = 2, c = 3;
int ans = 0;

// if (ans < 値) ans = 値; や ans = max(ans, 値); と書く手間を省き、可読性を高める
chmax(ans, a * 10 + b + c); // 12 > 0 なので ans は 12 に更新
chmax(ans, b * 10 + a + c); // 24 > 12 なので ans は 24 に更新
chmax(ans, c * 10 + a + b); // 33 > 24 なので ans は 33 に更新

// 結果: ans は 33 となる
```

## 92. 二次元いもす法: 矩形領域の高速な加算
2次元グリッド上の長方形領域に対する一律の加算を、毎回すべて塗りつぶすのではなく「4つの角（頂点）」への加減算のみで記録し、最後に全体を累積和で復元することで劇的に高速化する手法。

- 概要: 愚直に領域内すべてに $+1$ すると計算量が $O(N \times \text{面積})$ になるが、いもす法なら記録が $O(N)$、復元が $O(\text{縦} \times \text{横})$ となり、面積の大きさや操作回数 $N$ に依存せず高速に処理できる。
- 数学的イメージ: 領域 $[a, b) \times [c, d)$ への一様加算を、二階差分である4点 $(a, c), (a, d), (b, c), (b, d)$ へのディラックのデルタ関数的な重み付け $\{+1, -1, -1, +1\}$ に帰着させ、後から横・縦の二重積分（累積和）で元の面を復元する操作。

### 実装例：二次元いもす法による矩形の重なり計算
```cpp
// 1. 記録フェーズ: 矩形 [a, b) x [c, d) の4隅に +1 と -1 を配置 (計算量: O(N))
for (int i = 0; i < n; i++) {
    imos[a][c]++; // 左上（開始）
    imos[a][d]--; // 右上（終了・打ち消し）
    imos[b][c]--; // 左下（終了・打ち消し）
    imos[b][d]++; // 右下（過剰な打ち消しを補正）
}

// 2. 復元フェーズ: 累積和をとって面に展開する (計算量: O(H * W))
// まず横方向に累積和をとる
for (int i = 0; i < 101; i++) {
    for (int j = 0; j < 100; j++) {
        imos[i][j+1] += imos[i][j];
    }
}
// 次に縦方向に累積和をとる
for (int i = 0; i < 100; i++) {
    for (int j = 0; j < 101; j++) {
        imos[i+1][j] += imos[i][j];
    }
}
```

## 95. 逆順列の生成: Q[P[i]] = i と計算量比較
順列 $P$ において、「値 $x$ が元々どこにあったか」を示す逆順列 $Q$ を求める手法。ご提示いただいたインデックスを直接叩くコードの方が、`pair` を作ってソートする手法よりも**圧倒的に高速**である。

- 概要: ご提示のコードは時間計算量 $O(N)$ で完了するのに対し、`std::pair` を用いて `{値, 元のインデックス}` をソートするアプローチは $O(N \log N)$ かかるため。
- 数学的イメージ: 全単射（置換） $P: \{0, \dots, N-1\} \to \{0, \dots, N-1\}$ に対して、その逆写像 $P^{-1}$（すなわち $Q$）を $Q(P(i)) = i$ という関係式から直接構成する操作。

### 実装例：O(N) の直接構築 vs O(N log N) のソート
```cpp
// 1. 今回のコード（推奨: 圧倒的に速く、O(N)）
vector<int> Q(N);
for (int i = 0; i < N; ++i) {
    Q[P[i]] = i; // P[i] という値が、元の配列の i 番目にあったことを直接記録
}

// 2. pair を用いるアプローチ（非推奨: O(N log N) かかる）
// 順列でない一般の配列（値の重複や抜けがある場合）の「座標圧縮」などではこの手法が必要になるが、
// 1からNまでの順列であることが確定している場合は、ソートの計算量が無駄になる。
vector<pair<int, int>> A(N);
for (int i = 0; i < N; ++i) {
    A[i] = {P[i], i}; 
}
sort(A.begin(), A.end()); 
// A[i].second を取り出していくことになる
```

## 96. 平均値最大化の定石: 分母を払う式変形
「平均値」や「割合」の最大化を求める問題において、二分探索を用いて判定問題に帰着させ、不等式の分母を払って各要素のスコアの和にするテクニック。天才的な閃きのように見えるが、実は競技プログラミングにおける超有名な「定石（パターン）」である。

- 概要: 毎回ゼロから発想するわけではなく、「平均値や割合を最大化しろ」と言われた瞬間に「二分探索をして分母を払うあのパターンだな」と条件反射で式を組み立てるのが一般的。一度知ってしまえば強力な武器になる。
- 数学的イメージ: 集合 $S$ に対する有理関数の最大化問題 $\max \frac{\sum_{i \in S} V_i}{\sum_{i \in S} C_i}$ を、パラメータ $K$ を用いた判定問題 $\frac{\sum V_i}{\sum C_i} \ge K$ へと変換し、$\sum_{i \in S} (V_i - K \cdot C_i) \ge 0$ という線形結合の評価に帰着させる操作。

### 実装例：一般的な「平均値最大化」の判定ロジック
```cpp
// 価値 V[i]、コスト C[i] のアイテムからいくつか選び、
// 単位コストあたりの価値の平均 (ΣV / ΣC) を K 以上にできるか？

bool check(double K, const vector<double>& V, const vector<double>& C) {
    int N = V.size();
    vector<double> score(N);
    
    // 1. 各要素に対して「価値 - K * コスト」という新しいスコアを割り当てる
    // 割り算の式変形により、各要素を独立した足し算として扱えるようになる
    for (int i = 0; i < N; ++i) {
        score[i] = V[i] - K * C[i];
    }
    
    // 2. あとはこの score を使って、「合計が 0 以上になる選び方があるか？」を判定するだけ
    // （今回の問題は累積和を用いたが、問題によってはソートして貪欲に選んだり、DPを使ったりする）
    
    // 関連: [判定問題における単調性と二分探索の更新方向](#98-判定問題における単調性と二分探索の更新方向)
    return /* 条件を満たす選び方のスコア最大和 >= 0 */;
}
```
# 平均値最大化問題（二分探索＋累積和）のデバッグ備忘録

条件を満たす部分列の平均値（または割合）を最大化する問題において、実装時に陥りやすい代表的なバグと、その根本的な原因のまとめ。

## 1. 二分探索の構造に関する罠

### ① ループ内の状態初期化忘れ
- **症状**: 2回目の探索以降で、前回の探索時のスコアが残ってしまい判定が狂う。
- **原因**: 累積和の合計値 `sum` や最小値 `min_val` などの変数を `while` ループの外で定義・初期化し、ループ内でリセットしていなかったため。
- **対策**: 毎回の探索（判定関数の先頭）で、必ず状態管理の変数を初期値にリセットする。

### ② 探索範囲（境界）の更新方向の逆転
- **症状**: 答えが極端に小さくなる、または無限ループに陥る。
- **原因**: 条件を満たした（Yes）ときに「より厳しい目標（大きな値）」を探すべきなのに、逆に上限を下げてしまっていた（`rig = mid`）。
- **対策**: 最大化問題では「達成可能なら、そこが下限になる」と考えて `lef = mid` とする。単調性を意識する。

## 2. 累積和と状態更新に関する罠

### ③ 累積和の最小値の初期値ミス
- **症状**: 長さ1や2の区間が正しく評価されない、全体的にスコアがずれる。
- **原因**: 過去の累積和の最小値 `min_val` の初期値を `a`（配列の長さ）にしてしまっていた。
- **対策**: 「0個の要素を選んだときの和は0」であるため、累積和の最小値の初期値は必ず `0` にする。

### ④ 判定と更新の順序間違い（時系列の崩壊）
- **症状**: 要素数0の空区間を「条件達成」と誤認してしまう。
- **原因**: ループ内で `min_val` を更新した**直後**に `sum - min_val` を判定したため、自分自身（現在のスコア）を引き算してしまい、常に評価値が 0 になっていた。
- **対策**: 必ず「過去の最小値を使って判定」してから、「未来のインデックスのために最小値を更新」する。時系列を厳守する。

### ⑤ 配列の範囲外アクセス（Off-by-oneエラー）
- **症状**: 実行時エラー、または最後の文字の判定がおかしくなる。
- **原因**: `for (int i = 0; i < a + 1; ++i)` のように、文字列の長さ `a` に対して `a` 番目のインデックス（終端文字 `\0`）までアクセスしてしまった。
- **対策**: 文字列や配列のインデックスは `0` から `a - 1` まで。ループ条件は `i < a` を徹底する。

## 3. 数学的な制約に関する罠（最大の難所）

### ⑥ 式変形に伴う「隠れた前提条件」の無視
- **症状**: サンプルは合うのに提出するとWAになる。あり得ないほど高い平均値が出力される。
- **原因**: 不等式 $\frac{x - 2}{|t| - 2} \ge K$ の分母を払って $x - K|t| \ge 2 - 2K$ とする際、**「分母 $|t| - 2$ が正である（つまり $|t| \ge 3$）」**という前提条件を忘れ、長さ1や2の区間も含めて探索してしまったため（負の数を掛けると不等号が逆転するルールに違反した）。
- **対策**: 数式を変形した際は「分母の符号」に注意し、必要に応じて「長さ $L$ 以上の区間のみを判定対象とする」ように、累積和の比較対象（左端のインデックス）に制約を設ける。

## 97. 最短経路アルゴリズム: Dijkstra法 と Warshall-Floyd法
グラフ上の2点間を結ぶ経路のうち、辺の重み（コスト）の総和が最小となる経路を求めるアルゴリズム。目的に応じて「単一始点（1つのスタートから全部）」か「全点対間（すべての組み合わせ）」かを使い分ける。

### 97-1. ダイクストラ法 (Dijkstra's Algorithm)
- **目的**: 1つのスタート地点から、他のすべての頂点への最短距離を求める（単一始点最短経路）。
- **制約**: 辺の重みが **すべて0以上**（マイナスのコストがない）であること。
- **計算量**: $O((E + V) \log V)$ （※ $V$は頂点数、$E$は辺数。優先度付きキューを使用した場合）
- **数学的イメージ**: 始点から距離順に同心円状に探索範囲を広げる貪欲法。未確定の頂点集合 $U$ から暫定距離 $d(v)$ が最小の頂点 $u$ を確定させ、その隣接頂点 $v$ に対して $d(v) \leftarrow \min(d(v), d(u) + w(u, v))$ と緩和（Relaxation）を行う操作。

```cpp
// Dijkstra法の実装例 (O((E+V) log V))
// グラフ G は vector<vector<pair<int, long long>>> (行き先, コスト)
vector<long long> dist(N, 1e18); // 初期距離は無限大(1e18)
priority_queue<pair<long long, int>, vector<pair<long long, int>>, greater<>> pq;

dist[start] = 0;
pq.push({0, start}); // {距離, 頂点}

while (!pq.empty()) {
    auto [d, u] = pq.top();
    pq.pop();
    
    if (dist[u] < d) continue; // すでに短い経路で見つかっていればスキップ（枝刈り）
    
    for (auto& edge : G[u]) {
        int v = edge.first;
        long long cost = edge.second;
        if (dist[v] > dist[u] + cost) {
            dist[v] = dist[u] + cost;
            pq.push({dist[v], v});
        }
    }
}
```

### 97-2. ワーシャル・フロイド法 (Warshall-Floyd Algorithm)
- **目的**: すべての頂点から、他のすべての頂点への最短距離を一気に求める（全点対間最短経路）。
- **制約**: マイナスの重みをもつ辺があっても動作する（ただし、一周するとコストが無限に減る「負の閉路」がないこと）。
- **計算量**: $O(V^3)$ （※ 頂点数 $V$ が 400 程度までなら実用的に動く）
- **数学的イメージ**: 「頂点 $i$ から $j$ へ行く最短経路」を、「頂点 $k$ を経由した方が近いか？」という漸化式 $d_{i,j}^{(k)} = \min(d_{i,j}^{(k-1)}, d_{i,k}^{(k-1)} + d_{k,j}^{(k-1)})$ を用いて更新していく動的計画法（DP）。

```cpp
// Warshall-Floyd法の実装例 (O(V^3))
// dist[i][j] にはあらかじめ i から j への直接の辺のコストを入れておく（辺がなければ1e18, i==jは0）

// 経由地 k を一番外側のループにするのが最大のポイント！
for (int k = 0; k < N; ++k) {
    for (int i = 0; i < N; ++i) {
        for (int j = 0; j < N; ++j) {
            // iからjへ行くのに、kを経由した方が近ければ更新 (※オーバーフローに注意)
            if (dist[i][k] != 1e18 && dist[k][j] != 1e18) {
                dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]);
            }
        }
    }
}
```

### 97-3. ベルマン・フォード法 (Bellman-Ford Algorithm)
ダイクストラ法では対応できない「マイナスのコストを持つ辺」が含まれるグラフにおいて、単一始点最短経路を求めるアルゴリズム。さらに、辿るたびにコストが無限に減り続ける「負の閉路（Negative Cycle）」の検出も可能。

- **目的**: 1つのスタート地点から他のすべての頂点への最短距離を求める。負の閉路の検出。
- **制約**: なし（マイナスの辺があっても正常に動作する）。
- **計算量**: $O(V \times E)$ （※ ダイクストラ法より遅いため、マイナスの辺がない場合はダイクストラ法を優先する）
- **数学的イメージ**: 任意の最短経路は最大でも $V-1$ 本の辺からなるという性質に基づき、すべての辺に対する緩和操作 $d(v) \leftarrow \min(d(v), d(u) + w(u, v))$ を $V-1$ 回反復（漸化式の評価）する。$V$ 回目の反復でも更新が発生した場合、有界性を持たない（$\inf$ に発散する）負の閉路が存在すると判定する。



```cpp
// Bellman-Ford法の実装例 (O(V * E))
struct Edge {
    int u, v;
    long long cost;
};

int V, E;
vector<Edge> edges;
vector<long long> dist(V, 1e18); // 初期距離は無限大

dist[start] = 0;
bool has_negative_cycle = false;

// 1. 全ての辺に対する緩和を V-1 回繰り返す
for (int i = 0; i < V - 1; ++i) {
    for (int j = 0; j < E; ++j) {
        int u = edges[j].u;
        int v = edges[j].v;
        long long cost = edges[j].cost;
        
        // 始点から到達可能な頂点のみ更新を行う
        if (dist[u] != 1e18 && dist[u] + cost < dist[v]) {
            dist[v] = dist[u] + cost;
        }
    }
}

// 2. V回目の反復で負の閉路を検出する
// まだ更新できる辺がある場合、マイナス無限大に落ちる経路が存在する
for (int j = 0; j < E; ++j) {
    int u = edges[j].u;
    int v = edges[j].v;
    long long cost = edges[j].cost;
    if (dist[u] != 1e18 && dist[u] + cost < dist[v]) {
        has_negative_cycle = true;
        break;
    }
}
```
