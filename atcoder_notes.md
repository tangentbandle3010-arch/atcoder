# C++ & AtCoder 学習ノート

## 1. 文字列検索: find
文字列 $t$ の中から部分文字列 $s$ を探す関数。

- **使い方:** `t.find(s)`
- **戻り値:**
    - 見つかった場合：その開始位置のインデックス（$0, 1, 2, \dots$）
    - 見つからない場合：[npos](#2-不一致のシグナル-npos) を返す

### 実装例：隣り合う判定
```cpp
string t = "ACEGBDFAC";
string s; cin >> s;

// 見つかればYes、nposならNoを出力
cout << (t.find(s) != string::npos ? "Yes" : "No") << endl;
```

---

## 2. 不一致のシグナル: npos
`find` メソッドなどが「何も見つからなかった」時に返す特別な定数。

- **定義:** `static const size_t npos = -1;`
- **正体:** `size_t` 型（符号なし整数）の最大値。
- **注意点:** `int` 型の変数で戻り値を受け取ると、正しく比較できない場合がある。
- **参照:** [find関数の戻り値](#1-文字列検索-find)として利用される。

## 3. 特定の条件の要素を消す: erase-remove
`std::vector` などから、特定の条件に合う要素を「効率的に」削除する定番の手法。

- **使い方:** `v.erase(remove(v.begin(), v.end(), value), v.end());`
- **数学的イメージ:** 1. `remove`: 消したい要素を末尾に追いやり、生きてる要素だけの「新しい境界線」のイテレータを返す（写像の像を左に寄せる操作）。
  2. `erase`: 境界線から末尾までの「ゴミ」を物理的に削除する。

### 実装例：値が 3 の要素をすべて消す
```cpp
vector<int> v = {1, 3, 2, 3, 4};
// 3を末尾に寄せて、ゴミの開始地点から末尾まで削除
v.erase(remove(v.begin(), v.end(), 3), v.end());
// 結果: {1, 2, 4}
```

## 要素の削除と末尾への追加: erase / push_back
std::vector などのシーケンスコンテナにおいて、指定した位置の要素を物理的に削除する機能（erase）と、コンテナの末尾に新しい要素を追加する機能（push_back）。これらを組み合わせることで、配列全体をシフトするような操作が可能になる。

- 使い方: a.erase(a.begin()); / a.push_back(0);
- 数学的イメージ: 有限数列 $A = (a_0, a_1, \dots, a_{n-1})$ に対し、特定の元を取り除いて添字を詰める操作（erase）と、末尾に新たな元 $x$ を結合して $A' = (A, x)$ を構築する操作（push_back）。

### 実装例：先頭の要素を消し、末尾に0を追加する
```cpp
vector<int> a = {10, 20, 30, 40};

// 先頭の要素(10)を削除。残りの要素は自動的に前へ詰められる
a.erase(a.begin()); 

// 末尾に新しい値(0)を追加
a.push_back(0); 

// 結果: {20, 30, 40, 0}
```

## 4. キーと値の対応付け: std::map による頻度集計
連想配列（map）を用いて、特定の要素が何回現れたかを記録・管理する手法。

- 概要: 文字や文字列などの「キー」に対して、出現回数という「値」を紐付けて保持し、動的に更新する。
- 数学的イメージ: 写像 $f: A \to \mathbb{Z}$ において、各要素 $x \in A$ に対応する値 $f(x)$ を逐次加算し、集合 $A$ 上の頻度分布を構築する操作。

### 実装例：文字列に含まれる各文字の頻度を数え上げる
```cpp
string s;
cin >> s;
map<char, int> mp;

// 文字を走査し、対応する整数の値をインクリメント
for (auto &nx : s) {
    mp[nx]++;
}
```

## 5. 順列の全列挙: next_permutation
与えられた要素の並び（順列）を辞書順で「次のパターン」に並べ替える関数。

- 概要: 現在の範囲を辞書順で一つ後ろの順列に変換する。全パターンを網羅するには、事前に範囲を [sort](#sort) して最小の順列にしておく必要がある。
- 数学的イメージ: 有限集合 $S$ の置換 $n!$ 通りを、辞書式順序という全順序に基づいて全走査する操作。

### 実装例：異なる順列をすべて生成し、その総数を求める
```cpp
string S = "aba";
sort(S.begin(), S.end()); // 辞書順最小から開始するためのソート
set<string> s;

do {
    s.insert(S); // 現在の順列を集合に記録
} while (next_permutation(S.begin(), S.end()));

// 結果: s.size() は重複を除いた順列の総数
```

## 6. 最低1回の実行を保証する: do-while文
条件判定をブロックの「末尾」で行うことで、中身の処理を必ず1回は実行させる制御構文。

- 使い方: do { /* 処理 */ } while (条件式);
- 数学的イメージ: 状態 $s$ に対する写像 $f$ の適用を先行させ、遷移後の状態 $f(s)$ が継続集合 $A$ に属するかを事後判定する反復。

### 実装例：[next_permutation](#5) との典型的な併用
```cpp
string s = "abc";
do {
    // 最初の状態（"abc"）を処理した後に、次の順列へ遷移させる
    cout << s << endl;
} while (next_permutation(s.begin(), s.end()));
// 判定が後ろにあるため、最初の並びも漏らさず処理できる
```

## 7. データのペア管理と比較: std::pair
2つの関連する値を一つの「組」としてまとめ、それらを一括で管理・比較する手法。

- 概要: `pair<T1, T2>` を用いて日付や座標などの関連データを一つに束ね、`vector` などのコンテナに格納して走査や検索を行う。値へのアクセスは p.first, p.second
- 数学的イメージ: 直積集合 $M \times D$ の要素 $(m, d)$ を定義し、入力された要素が特定のラベル集合 $G \subset M \times D$ に属するか（$(m, d) \in G$）を判定する操作。

### 実装例：特定のペアが含まれているか検索する
```cpp
int m, d;
cin >> m >> d;

// pair の vector を定義し、あらかじめ対象の集合を用意する
vector<pair<int, int>> gosekkulist = {
    {1, 7}, {3, 3}, {5, 5}, {7, 7}, {9, 9}
};

bool isgosekku = false;
for (const auto &date : gosekkulist) {
    // make_pair や {m, d} で一時的なペアを作り、== で直接比較可能
    if (make_pair(m, d) == date) isgosekku = true;
}

// 判定結果の出力
cout << (isgosekku ? "Yes" : "No") << endl;
```

### 実装例：値と元のインデックスを紐付けて最大値の場所を特定する
```cpp
vector<int> a = {30, 10, 50, 20};
vector<pair<int, int>> b(a.size());

// 第一成分に値、第二成分に元のインデックス（1-based）を持つ元を生成
for (int i = 0; i < a.size(); i++) {
    b[i] = {a[i], i + 1};
}

// 辞書式順序に従いソート（第一成分で比較し、同値なら第二成分で比較）
sort(b.begin(), b.end()); // 関連: [要素の並び替え: sort](#sort)

// 最大値（末尾の元）の第二成分（元のインデックス）を取得
int max_index = b.back().second;
// 結果: 3 (値50のインデックス)
```

## 8. 末尾から検索する：rfind
文字列の中から、指定した文字や文字列が「最後に現れる位置」を検索する機能。

- 使い方: `s.rfind(target)`
- 数学的イメージ: 文字列をインデックスの有限集合 $I$ としたとき、条件を満たすインデックス $i \in I$ の最大値を返す写像。

### 実装例：特定の文字が最後に現れる位置を求める
```cpp
string s = "abracadabra";
// 'a' を末尾から検索（最後に見つかるのはインデックス 10）
auto pos = s.rfind('a');

if (pos != string::npos) {
    // 見つかった場合、そのインデックスを表示
    cout << pos << endl; 
} else {
    // 見つからなければ [npos](#npos) と比較して判定
    cout << -1 << endl;
}
```

## 9. 1-indexed と配列の余白
競技プログラミングなどで、問題文の「1番目」という指定をそのまま配列の添字として扱うために、サイズを少し大きく確保する手法。

- 使い方: `int a[N + 1][M + 1];` （$N \times M$ の行列を扱う場合）
- 数学的イメージ: 集合 $S$ の添字集合 $I = \{0, 1, \dots, n\}$ において、部分集合 $\{1, \dots, n\}$ のみを実運用に割り当てる写像。

### 実装例：2行2列の入力を1ベースで受け取る
```cpp
int r, c;
// 2x2なら a[3][3] で足りるが、余裕を持って a[4][4] とすることもある
int a[4][4]; 

for(int i = 1; i <= 2; i++) {
    for(int j = 1; j <= 2; j++) {
        // 0番目を無視して 1番目, 2番目に直接格納
        cin >> a[i][j];
    }
}
// 入力が (1, 2) なら a[1][2] をそのまま参照できる
cout << a[r][c] << endl;
```

## 10. 文字を小文字に変換する: tolower
英大文字を対応する小文字に変換する機能。アルファベット以外（数字や記号）や既に小文字である場合は、元の値をそのまま返す。

- 数学的イメージ: 写像 $f: C \to C$。大文字の集合を $U \subset C$、小文字の集合を $L \subset C$ とすると、$x \in U$ のとき $f(x) \in L$、それ以外は $f(x) = x$（恒等写像）。

### 実装例：先頭文字を小文字にして接頭辞を加える
```cpp
string s = "Apple";
// 先頭文字 'A' を小文字 'a' に変換
s[0] = tolower(s[0]);
// 文字列を結合
string ans = "an " + s;
// 結果: "an apple"
```

## 11. 特定の要素を別の値に置き換える: replace
std::string や std::vector などのコンテナ内で、特定の値に一致する要素をすべて新しい値に置換する手法。

- 使い方: replace(v.begin(), v.end(), old_value, new_value);
- 数学的イメージ: 特定の元 $a$ を $b$ に移し、それ以外を固定する写像。特定の2元の互換 $(a\ b)$ を実現する場合は、集合外の元 $x$ を経由した合成写像として適用する。

### 実装例：文字列内の '0' と '1' を反転（スワップ）する
```cpp
string s = "1001";
// 一時的な値 'x' を経由した合成写像により、'0' と '1' の互換を実現
replace(s.begin(), s.end(), '0', 'x');
replace(s.begin(), s.end(), '1', '0');
replace(s.begin(), s.end(), 'x', '1');
// 結果: "0110"
```

## 12. 範囲内の最大値の場所を特定する: max_element
指定された範囲の要素を調べ、最大値を持つ「最初の要素へのイテレータ（位置情報）」を取得する手法。

- 使い方: max_element(v.begin(), v.end());
- 数学的イメージ: 全順序集合 $(S, \le)$ において、任意の元 $x$ に対して $x \le m$ を満たす最大の元 $m$ を特定し、元の配列空間におけるその「座標（位置）」を返す写像。

### 実装例：最大値のインデックス（位置）を直接計算する
```cpp
vector<int> a = {10, 50, 20, 50, 30};

// 先頭イテレータとの距離（差分）を計算し、0-basedインデックスを取得
int index_0 = max_element(a.begin(), a.end()) - a.begin();
// 結果: 1 （同値の最大値が複数ある場合、最も先頭に近いものが選ばれる）

// 競技プログラミングで頻出の 1-based インデックスの取得
int index_1 = max_element(a.begin(), a.end()) - a.begin() + 1;
// 結果: 2

// 関連: 実体（値そのもの）が欲しい場合は * 演算子でデリファレンスする
// int max_val = *max_element(a.begin(), a.end());
```

---
### 💡 注釈: なぜ a.begin() を引く必要があるのか？
`max_element` などの標準アルゴリズムは、配列のインデックスではなく**「イテレータ（メモリ上の位置情報）」**を返す仕様になっているため。

- **仕組みのイメージ**: 
  イテレータは「メモリの住所」のようなもの。「最大値の住所」から「先頭の住所（`a.begin()`）」を引き算して、初めて先頭からの「距離」が求まる。
- **インデックスの取得**:
  1. `イテレータ - a.begin()` の計算結果が、そのまま **0-basedのインデックス**（0始まり）になる。
  2. 競技プログラミング等で **1-basedのインデックス**（1番目、2番目…）にする場合は、最後に `+ 1` をする。
- **注意点**:
  イテレータのままでは、数値として `cout` で出力することはできない（コンパイルエラーになる）。

## 13. 最も近い整数に丸める: round
浮動小数点数を最も近い整数に丸める（四捨五入する）手法。競技プログラミングでは、計算誤差を吸収するために微小な値（$\epsilon$）を加えてから処理するテクニックがよく使われる。

- 使い方: round(value); （戻り値は浮動小数点数型であるため、通常は int 等にキャストして用いる）
- 数学的イメージ: 実数 $x$ を最も近い整数に写す写像。半整数（小数部分がちょうど $0.5$）の場合は絶対値が大きくなる方向へ丸められる（$x \ge 0$ においては $\lfloor x + 0.5 \rfloor$ と同値）。

### 実装例：浮動小数点数の誤差を考慮した丸め処理
```cpp
double a = 2.4999; // 本来は 2.5 になるべき計算結果（誤差を含むと仮定）

// 通常の丸め（このままだと2.4999は2に丸められてしまう）
int normal_ans = (int)round(a); // 結果: 2

// 微小な値（0.0005など）を足すことで、誤差による意図しない切り捨てを防ぐ
int safe_ans = (int)round(a + 0.0005);
// 結果: 3
```

## 14. 特定の形式に従って入力を抽出する: scanf
指定したフォーマット（形式）文字列に従って標準入力からデータを読み取り、変数に直接代入するC言語由来の手法。競技プログラミングにおいて、特定の文字（`.` や `/` など）を区切りとして読み飛ばしながら数値を抽出する際に極めて有効。

- 使い方: scanf("フォーマット文字列", &変数1, &変数2, ...);
- 数学的イメージ: 入力された文字列の集合 $\Sigma^*$ に対し、フォーマット指定子が定義する構造とのパターンマッチングを行い、抽出した部分文字列を目的のデータ型の空間（例: $\mathbb{Z} \times \mathbb{Z}$）へ直接写す射影。

### 実装例：小数点を区切り文字として整数部と小数部を分割する
```cpp
int a, b;
// 入力例 "3.14" において、'.' を単なる区切り（無視する文字）として扱い、
// 3 を a に、14 を b に整数として抽出・代入する。
// ※代入先の変数はメモリアドレスを渡す必要があるため '&' が必須
scanf("%d.%d", &a, &b);

// 小数第一位の値を使って四捨五入を判定するロジックの例
// 浮動小数点数（double等）の丸め誤差を完全に回避できる利点がある
if (to_string(b)[0] >= '5') {
    ++a;
}
```
---
### 💡 注釈: これは scanf (C言語スタイル) 特有の手間ですか？
はい、その通りです。`char S[3];` のように手動で「文字数＋1」の余裕を計算・確保するのは、**C言語由来の関数（`scanf` や `printf` など）で文字配列を扱う際などに特有の書き方**です。

- **C言語スタイル (`char` 配列 + `scanf`)**:
  - コンピュータは「用意された箱（メモリ）」に直接データを流し込むだけなので、人間が事前に十分なサイズの箱を用意し、終端（ヌル文字 `\0`）のスペースまで管理する責任を持ちます。
- **C++スタイル (`std::string` + `cin`)**:
  - `string` という便利なクラスがメモリ管理を全自動で行ってくれます。
  - `string S; cin >> S;` と書くだけで、入力された長さに合わせて裏側で勝手に箱を広げ、ヌル文字の処理も隠蔽してくれるため、人間がサイズを気にする必要はありません。

競技プログラミングでは、極限の実行速度を求めてあえてC言語の `scanf` と `char` 配列を使う層もいますが、バグを減らしたい場合は `std::string` と `cin` を使うのが現代C++の基本スタイルです。

## 15. ビット演算で大文字・小文字を反転する: ^= 32
ASCIIコードの規則性を利用し、排他的論理和（XOR）を用いてアルファベットの大文字と小文字を高速に相互変換（反転）する手法。

- 使い方: c ^= 32;
- 数学的イメージ: アルファベットの文字集合に対する対合（インボリューション） $f(x) = x \oplus 32$ を適用し、大文字の集合と小文字の集合との間で双方向の全単射を構成する。

### 実装例：条件を満たしたときに先頭文字の大小を反転させる
```cpp
char S[3] = "Y";
char T[3] = "abc";

// Sの先頭が 'Y' であるか判定
if (S[0] == 'Y') {
    // ASCIIコードにおいて大文字と小文字はちょうど 32 (2^5) 離れている。
    // 32とのXOR演算により特定ビットのみを反転させ、'a' を 'A' に変換する
    T[0] ^= 32;
}
// 結果: T は "Abc" になる
// 関連: 入力の抽出 [scanf](#14)
```
---
### 💡 注釈: なぜ `^= 32` で大文字・小文字が反転するのか？
アルファベットの「ASCIIコード（コンピュータ内部での文字の背番号）」の2進数表現における美しい規則性を利用しているためです。

- 'A' の番号は 65 （2進数で `01000001`）
- 'a' の番号は 97 （2進数で `01100001`）

大文字と小文字の番号は**常に 32 離れています**。
これを2進数で見ると、右から6番目のビット（$2^5 = 32$ の位）が `0` なら大文字、`1` なら小文字という規則になっています。
XOR（排他的論理和 `^`）は「指定した箇所のビットだけを反転させる」という性質を持つため、`32`（2進数で `00100000`）とXOR演算をすることで、**文字の種類（1〜5番目のビット）は維持したまま、大文字・小文字のフラグだけを綺麗に切り替える**ことができるのです。

## 16. 特定の形式で値を出力する: printf
指定したフォーマット（形式）文字列の「穴（指定子）」に変数を埋め込み、標準出力へ書き出すC言語由来の手法。`cout` と比較して実行速度が速い場合が多く、競技プログラミングでも好まれる。

- 使い方: printf("フォーマット文字列", 変数1, 変数2, ...);
- 数学的イメージ: 文字列のテンプレート（型）に対して、変数の直積空間から値を代入し、具体的な出力文字列へと写す射影。

### 実装例：文字列を埋め込んで改行する
```cpp
char T[] = "Abc";

// %s は「文字列（char配列など）」を埋め込むためのフォーマット指定子
// \n は「改行」を表す特殊文字
printf("%s\n", T);
// 結果: Abc と出力された後、改行される

// 関連: フォーマットに従った入力 [scanf](#14)

```

## 17. 重複の排除と種類数の判定: std::set
複数の値から重複を自動的に排除し、異なる要素が「何種類あるか」を簡潔に判定する手法。

- 使い方: set<int> S({A, B, C});
- 数学的イメージ: 与えられた多重集合から重複元を縮約して純粋な集合 $S$ を構築し、その濃度（要素数） $|S|$ を評価する。

### 実装例：3つの変数が「ちょうど2種類」であるかを判定
```cpp
int A = 5, B = 7, C = 5;
// 波括弧 {} で初期化。重複する 5 は自動的に1つにまとめられる
set<int> S({A, B, C}); 

// 要素数を確認する
if (S.size() == 2) {
    // 同じ値のペアが1組だけ存在し、残りが異なる値の場合の処理
}
```

## 18. 集合への要素の追加: insert
std::set などの連想コンテナに新しい要素を追加する手法。すでに同じ値が存在する場合は自動的に無視される。

- 使い方: s.insert(value);
- 数学的イメージ: 既存の集合 $S$ に新たな元 $x$ を追加し、和集合 $S \cup \{x\}$ を構築する。すでに $x \in S$ である場合、集合の構成と濃度は変化しない。

std::vector などのシーケンスコンテナにおいて、イテレータを用いて指定した特定の位置（インデックス）に新しい要素を割り込ませる手法。

- 使い方: v.insert(v.begin() + K, X);
- 数学的イメージ: 有限数列 $A = (a_0, a_1, \dots, a_{n-1})$ の第 $k$ 添字位置に元 $x$ を挿入し、要素が右へシフトした新たな数列 $A' = (a_0, \dots, a_{k-1}, x, a_k, \dots, a_{n-1})$ を構築する操作。

### 実装例：値を1つずつ追加して種類数を数える
```cpp
set<int> s;

s.insert(5);
s.insert(7);
s.insert(5); // 既に 5 が存在するため追加されない

// 結果: s の要素は {5, 7} のみとなる
if (s.size() == 2) {
    // 追加された一意な値がちょうど2種類だった場合の処理
}
```

### 実装例：配列のインデックス K の位置に値 X を挿入する
```cpp
vector<int> A = {10, 20, 30};
int K = 1;  // 挿入したいインデックス
int X = 15; // 挿入する値

// 先頭イテレータに K を足して場所を指定する
A.insert(A.begin() + K, X);
// 結果: {10, 15, 20, 30}
```

## 19. 浮動小数点数の出力フォーマット指定: fixed / setprecision
実数（浮動小数点数）を出力する際に、指数表記を防ぎ、小数点以下の表示桁数を固定するためのマニピュレータ。

- 使い方: cout << fixed << setprecision(桁数) << 値;
- 数学的イメージ: 実数 $x \in \mathbb{R}$ を指定された非負整数 $n$ の精度空間に射影し、小数第 $n$ 位で丸められた近似値の文字列表現へ変換する写像。

### 実装例：割り算の結果を小数点以下3桁で出力する
```cpp
double T = 10.0, X = 3.0;
// 指数表記を無効化（fixed）し、小数点以下を3桁に固定（setprecision）
cout << fixed << setprecision(3) << T / X << "\n";
// 結果: 3.333

```

## 20. 符号付き整数型とマクロ置換の回避: signed
単独で記述された場合は通常の `int`（符号付き整数）と完全に同義となる型指定子。競技プログラミングにおいて `#define int long long` というオーバーフロー対策のマクロを組む際、標準規格で戻り値が `int` でなければならない `main` 関数のコンパイルエラーを回避する目的で、置換対象にならない `signed main()` として使われる（提示コードではそのテンプレートの名残として書かれている）。

- 使い方: signed main() { ... }
- 数学的イメージ: 整数環 $\mathbb{Z}$ の部分集合である有限の符号付き整数区間 $[-2^{31}, 2^{31}-1]$ への写像を、字句置換系（マクロ）から独立して束縛する。

### 実装例：int型の長整数化を伴う環境での利用
```cpp
#define int long long // オーバーフローを防ぐ競プロ特有の記述

// int main() にすると long long main() に置換されエラーになるため signed を使う
signed main() {
    // 関数内の int はすべて long long として評価される
    int large_value = 1e18; 
}
```

## 21. 文字列の長さの取得: length / size
std::string 型の変数が保持する文字列の長さ（文字数）を取得するメソッド。結論から言うと、`length()` と `size()` の機能は「全く同じ」である（前者は文字列としての直感的な表現、後者は他のSTLコンテナとの統一性を保つために両方用意されている）。

- 使い方: S.length(); または S.size();
- 数学的イメージ: 文字列を文字の有限列 $S = (c_0, c_1, \dots, c_{n-1})$ と見なしたとき、その列の長さ（要素の濃度） $n = |S|$ を評価する。

### 実装例：文字列の長さの取得と末尾の文字の判定
```cpp
string S = "bus";

// length() と size() は完全に等価（どちらを使ってもよい）
int len = S.length(); // 結果: 3
int sz = S.size();    // 結果: 3

// 長さを利用して末尾の文字（インデックスは len - 1）にアクセスする
if (S[len - 1] == 's') {
    S += "es";
} else {
    S += "s";
}
// 結果: "buses"

```

## 22. 高速なハッシュ集合と直接構築: unordered_set / emplace
要素の順序を保持しない代わりに、追加や検索をハッシュ表を用いて平均定数時間（非常に高速）で行うコンテナと、その挿入メソッド。

- 使い方: A.emplace(x);
- 数学的イメージ: 普遍集合から元 $x$ をハッシュ関数 $h(x)$ によってテーブルに写像し、順序を持たない和集合 $S \cup \{x\}$ を構築する。

### 実装例：N個の数値を高速な集合に格納する
```cpp
unordered_set<int> A;
int N = 3;

// N回繰り返す（ここでは標準的なfor文で記述）
for (int i = 0; i < N; ++i) {
    int x = 5; // 実際は cin >> x などで入力を受け取る
    // insertの代わりにemplaceを使うことで、コンテナ内に直接要素を構築して追加する
    A.emplace(x); 
}
```
